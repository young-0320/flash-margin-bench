# 5. PRBS — 정답지를 안 보내고 채점하는 법 (측정 패턴 사양)

- 관련 결정: 계약 19 (패턴 = PRBS-15, 시드 = {1, index[13:0]})

## 왜 패턴 "사양"이 필요한가

에러 1개의 정의는 "받은 비트 ≠ 기대한 비트"다. 채점이 성립하려면 보내는 쪽과 채점하는 쪽이 **같은 정답지를 공유**해야 한다. 그리고 그 정답지의 내용 자체가 측정의 품질을 좌우한다.

시험지 비유로 나쁜 정답지 두 가지:

- **올-제로 (`0000...`)**: 모든 문제의 답이 "0"인 시험지. 답안지를 한 칸 밀려 써도(타이밍이 틀어져 옆 비트를 읽어도) 만점이 나온다. 고장이 에러로 번역되지 않으니 **욕조 곡선의 벽 자체가 안 생긴다.**
- **교대 (`0101...`)**: 한 칸 밀림은 잡지만 주기가 2라서 **두 칸 밀리면 다시 만점** — 짝수 슬립이 전부 투명해진다.

이상적인 정답지는 "어느 칸으로 밀려 써도 확률 1/2로 틀리는" **무작위 비트열**이다. 그러면 윈도우 밖에서 BER이 0.5로 포화하는데, 이것이 몬테카를로와 `bathtub_analysis.py`가 처음부터 가정해 온 성질이다. 그런데 진짜 난수는 채점자가 정답을 모르게 된다. 여기서 PRBS가 등장한다.

## PRBS: 무작위처럼 보이는 결정론

PRBS(Pseudo-Random Binary Sequence)는 **시프트 레지스터와 XOR 게이트 하나**로 만드는 비트열이다(이 회로를 LFSR이라 부른다). 매 클럭마다: 레지스터의 특정 두 칸을 XOR한 결과를 맨 앞에 밀어 넣고, 맨 뒤에서 한 비트를 출력한다. 동전 던지기처럼 보이지만 완전히 결정론적이라, **같은 시작값(시드)에서 돌리면 언제 어디서든 똑같은 수열**이 나온다.

즉 정답지를 통째로 전달하는 대신 **"생성 규칙 + 시작값"만 공유**한다. FPGA의 패턴 생성기, 비교기, 파이썬 golden model이 각자 15개 플립플롭(또는 코드 몇 줄)으로 같은 정답을 독립 재생성한다 — 오라클 독립성까지 공짜로 따라온다.

## 엔지니어링 수치 — 왜 하필 15인가

우리 사양: **다항식 x¹⁵ + x¹⁴ + 1** (15비트 레지스터, 14·15번 칸을 XOR — 통신 표준에서 검증된 최대 길이 구성).

| 항목                | 값                                   | 의미                                                                     |
| ------------------- | ------------------------------------ | ------------------------------------------------------------------------ |
| 주기                | 2¹⁵ − 1 =**32,767비트**     | 버스트 2,048비트는 주기의 6.25% →**버스트 안에서 반복 없음**      |
| 하드웨어 비용       | FF 15개 + XOR 1개                    | 7Z020 FF 106,400개의 0.014% — 사실상 공짜                               |
| 0/1 균형            | 한 주기에 1이 16,384개, 0이 16,383개 | BER 0.5 포화의 전제인 균형성                                             |
| 이웃 비트 상이 확률 | ≈ 1/2 (한 주기에 전이 16,384회)     | k비트 슬립 시(k = 1..2,047 어느 값이든) BER ≈ 0.5 — 모든 슬립이 가시화 |
| 최장 런             | 1이 15개 / 0이 14개 연속             | 저주파 쏠림 한계 — DC 균형 유지                                         |

**PRBS-7이면 안 되는 이유**: 주기가 2⁷−1 = 127비트라 버스트 2,048비트 안에 약 16번 반복된다. 그러면 **127의 배수만큼 밀린 슬립은 정답과 다시 일치**해 투명해진다 — 교대 패턴의 "짝수 슬립 구멍"과 같은 병이 주기만 길어진 채 남는 것. PRBS-15는 버스트 내 무반복이라 이 구멍이 없다. PRBS-23(주기 838만)은 이득 없이 레지스터만 커진다 — 버스트 무반복이면 충분하다.

전기적으로도 전이 밀도 ~50%는 실제 데이터와 비슷한 스펙트럼을 만들어, 심벌 간 간섭 같은 실사용 스트레스를 자극한 채로 마진을 잰다 — 얌전한 패턴으로 재면 마진이 낙관적으로 부풀 수 있다.

## 시드 규약 

시드는 15비트인데 **0만은 금지**다: 올-제로 상태에서 XOR 피드백은 영원히 0만 낳는다(락업). 우리 규약은 상위 비트를 1로 고정하고 하위 14비트에 인덱스를 넣는다:

$$
seed = \{1,\ index[13:0]\}
$$

- 구조적으로 0이 될 수 없어 락업 방어에 조건문이 필요 없다.
- **index = 루프백: 스텝 내 읽기 번호(0..N−1)** — 모든 위상 스텝이 동일한 자극 시퀀스를 받아, 스텝 간 차이가 순수하게 캡처 타이밍에만 귀속된다(통제 변인).
- **index = 실칩: 페이지 주소 하위 14비트** — "주소만 알면 그 페이지에 뭐가 써 있어야 하는지 안다"가 되어, 쓰기·읽기·채점 장부가 주소 하나로 닫힌다.
- 정직한 한계: W25Q64는 32,768페이지라 하위 14비트가 같은 페이지 쌍(16,384쌍)은 같은 시드를 공유한다. 시드의 목적은 페이지 유일성이 아니라 **버스트 단위 재생성 가능성**이므로 실험에 무해하다 — 스윕은 소수의 페이지만 쓴다.

## 버스트마다 재시드하는 이유

버스트 시작마다 시드를 다시 넣으면, 비교기는 **이전 이력 없이** 그 버스트의 정답을 그 자리에서 만들 수 있다. 도중에 에러가 나든 측정을 재시작하든 다음 버스트에서 동기가 저절로 회복된다. 이력에 의존하는 연속 스트림 방식이었다면 슬립 하나가 이후 전체를 오염시켰을 것이다 — 재시드는 오염의 전파를 버스트 하나로 격리하는 방화벽이다.
