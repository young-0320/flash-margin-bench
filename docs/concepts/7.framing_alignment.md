# 7. 프레이밍 정렬 — 받아쓰기는 완벽한데 채점이 밀리는 문제

- 관련 결정: 계약 §1 (샘플 클럭만 시프트) · 로그 7 (flash_rx 3후보) · 로그 11 (합성 프리앰블·PAY_LEAD)

## 쉬운 버전 — 자막 싱크와 딱따기

**문제는 오답이 아니라 밀림이다.** 외국 영화를 보는데 자막이 1초 밀려 있다고 하자. 대사가 전부 안 맞아 보이지만, 배우가 연기를 못한 것도 번역이 틀린 것도 아니다 — 싱크 문제다. 우리 측정도 똑같다. 수신기는 비트를 하나도 안 틀리고 받아 적는데, **정답지(PRBS)의 몇 번째 글자부터 대조할지**가 한 칸 밀리면 채점표에는 "반타작"(BER 0.5)이 찍힌다. 칩은 멀쩡한데 채점이 틀린 것.

**밀림은 고장이 아니라 이 실험의 필연이다.** 우리는 샘플 시계를 일부러 15.87ps씩 늦춰 가며 찍는다(그게 위상 스윕이다). 계속 늦추다 보면 어느 순간부터는 "이번 비트의 끝"이 아니라 "다음 비트의 시작"을 찍게 된다 — 채점 기준이 한 칸 밀리는 순간이 스윕 중간에 반드시 온다. 그러니 밀림을 없앨 수는 없고, **밀려도 점수가 안 깨지게** 만들어야 한다.

**해법: 싱크를 세 가지로 틀어 보고 제일 잘 맞는 걸 쓴다.** 자막 싱크가 애매하면 -1초 / 0초 / +1초 세 버전으로 대사를 맞춰 보면 된다. 싱크 문제였다면 셋 중 하나는 딱 맞고, 진짜 형편없는 영화였다면(칩이 진짜 틀렸다면) 어느 싱크로 봐도 엉망이다. flash_rx가 하는 일이 정확히 이것이다: 정렬 후보 3개로 동시에 채점하고 **최고 점수(min 에러)를 채택**한다. 진짜 에러는 싱크를 바꿔도 사라지지 않으므로 이 방법은 관대해지지 않는다.

**그럼 싱크의 기준점은 누가 주나 — 딱따기.** 영화 촬영에서 소리와 영상을 나중에 맞추려고 치는 딱따기(클래퍼보드)처럼, 본문 앞에 눈에 띄는 표식(프리앰블 1111)을 붙여 "여기서부터 세라"를 알린다. 여기서 루프백과 실칩이 갈린다:

- **루프백**: 딱따기 소리가 영상과 **같은 필름에** 녹음돼 있다(프리앰블이 페이로드와 같은 배선을 탄다). 필름이 얼마나 늦게 도착하든 둘이 같이 늦으니 싱크는 저절로 맞는다.
- **실칩**: 칩은 딱따기를 쳐 주지 않는다(SPI 칩은 데이터만 보낸다). 그래서 **우리가 옆에서 쳐 준다**(마스터가 "곧 데이터 온다"고 예고하면 수신 쪽에서 프리앰블을 합성). 이제 우리 딱따기와 칩의 데이터가 **서로 다른 길로** 도착하므로, 두 길의 시간차만큼 미리 당겨 쳐야 한다 — 그 "몇 박자 먼저"가 PAY_LEAD라는 상수다.

이 차이가 고장 났을 때의 증상까지 가른다: 루프백은 딱따기가 안 오면 채점이 시작을 못 해 **멈추고**(TIMEOUT), 실칩은 딱따기를 우리가 치니까 칩이 죽어도 **채점은 끝까지 돈다**(대량 에러로 완주). 아래 판독표의 핵심 비대칭이다.

## 엔지니어링 버전

**밀림의 정체 (mesochronous)**: clk_core와 clk_sample은 같은 VCO 출신 — 주파수 동일, 위상만 다르다. 드리프트가 없어 CDC는 준정적 규율(3번 문서)로 충분하지만, 샘플열의 **정수 비트 정렬 d가 위상의 함수**가 된다. 전이 경계 부근 위상에서는 경계 샘플이 0/1 어느 쪽으로도 읽혀 정렬 판정 자체가 버스트마다 ±1 흔들린다. 고정 정렬 비교 시 스윕 절반이 BER 0.5 절벽 — 분석기 체크 5가 잡는 서명이다.

**flash_rx의 3단 구조** (버스트마다 독립 — 5번 문서의 재시드처럼 오염 격리):

1. 갭의 0-run(≥8)으로 자격을 얻은 뒤 첫 1 샘플 = **t0** (프리앰블 비트 0)
2. 페이로드 시작 후보 **t0+3 / t0+4 / t0+5**를 PRBS 엔진 1개 + 지연 플롭 2개로 병렬 채점
3. **e_i = min(e0, e1, e2)**

min이 속지 않는 논거: 밀린 PRBS는 무상관이라 **틀린 정렬의 점수 ≈ B/2**, 옳은 정렬은 진짜 에러만 센다. 벽 적합 영역(BER ≤ 0.25)에서 항상 명확히 분리. 정직한 한계 — 창 밖 포화 영역에선 min(셋)이 0.5를 약간 밑돈다(통계가 쓰는 영역 밖, 로그 7).

**루프백 vs 실칩의 예산 구조**:

| 항목            | G0 루프백                        | 실칩 SPI                                     |
| --------------- | -------------------------------- | -------------------------------------------- |
| 프리앰블 출처   | TX가 페이로드 앞에, 같은 배선    | shim이 로컬 합성 (pay 예고 → 1111 + MISO 먹스) |
| 정렬 vs 경로 지연 | 무관 (표식·본문이 같이 밀림)   | 왕복 지연의 함수 (표식은 안 밀림)            |
| 후보 3칸의 용도 | 전부 위상 ±1 흡수               | 위상 ±1 + **왕복 지연의 정수 UI 부분**      |
| 보정 상수       | 불필요 (tb 지연 0.2~63ns 통과)   | **PAY_LEAD=5** (`flash_spi_ctrl.v` 헤더 산식) |

pay는 clk_core에서 출발해 2FF 동기로 clk_sample에 도착하고(+2~3), shim이 프리앰블 4비트를 붙인 뒤 MISO로 먹스한다. 표식 경로 지연(동기화기)과 본문 경로 지연(SCLK 왕복 + tCLQV + 캡처)의 **차**가 후보 창 {3,4,5} 중앙에 오도록 pay를 데이터보다 PAY_LEAD 사이클 먼저 올린다. 왕복 지연이 1 UI를 넘을 때마다 정렬 중심이 1칸 이동하므로: 25MHz(UI 40ns, 왕복 ~10ns)는 여유, **75MHz(UI 13.3ns)는 왕복이 1 UI에 근접 — PAY_LEAD 1 감소 재보정이 걸릴 수 있다** (사다리 체크 항목, 로그 11·12).

**후보 밖 시나리오 — "0.3칸"과 "±2칸"의 운명.** 소수점 밀림은 존재하지 않는다: 샘플러는 비트당 한 번 찍는 양자화기라, 위상을 0.3칸 밀면 수열이 밀리는 게 아니라 **각 비트의 어디를 찍는지**가 바뀔 뿐이다. 그 효과는 경계 근처 샘플의 불안정으로 나타나고 — 그게 바로 측정 대상인 욕조 벽이다. 프레이밍에 남는 흔적은 "첫 1 판정이 어느 칸으로 떨어지나"의 양자화 흔들림 하나이고, 이것이 정확히 ±1이라 후보가 3개다. 한편 정렬 중심이 통째로 창 밖(±2)으로 나가는 것은 루프백에선 구조적으로 불가능하고(표식·본문 동일 경로), SPI에선 고정 왕복 지연이 1 UI를 넘을 때 일어난다 — 이때 세 후보 전부 ≈ B/2가 되어 **전 위상 BER 0.5로 시끄럽게 죽는다**(가짜 곡선으로 둔갑 불가, 체크리스트가 기각). 후보를 5개로 늘리지 않는 이유: 흔들리는 양(±1)은 유계라 3개면 덮이고, 흔들리지 않는 양(고정 지연)은 보정 상수(PAY_LEAD)의 몫이며, 무상관 후보가 늘수록 min의 포화 점수가 0.5 아래로 끌려 5번 문서의 "창 밖 = 0.5 포화" 보장이 무뎌진다. **후보는 지터 몫, 상수는 지연 몫.**

## 고장 서명 판독표 — 브링업 실전용

| 관측                                       | 원인                                             | 조치                                            |
| ------------------------------------------ | ------------------------------------------------ | ----------------------------------------------- |
| **전 위상 BER ≈ 0.5** (바닥 없음)         | 정렬이 후보 창 밖 (지연 > 예산)                  | PAY_LEAD 재보정 — RTL 버그 아님                |
| 루프백 무신호 → **TIMEOUT**               | 표식이 안 와서 RX가 영원히 HUNT                  | 점퍼/배선 확인 (R10이 잡아줌)                   |
| SPI 무칩/단선 → **TIMEOUT 없이 완주 + 대량 에러** | 표식은 합성이라 항상 성립, 본문만 0 고정 (에러 수 = 기대 PRBS의 1 개수) | 배선/전원/CS 확인. **"측정은 되는데 칩 불량" 오독 금지** |
| 스텝 0에서 **phase_pos_mismatch**          | ELF만 재로드 (MMCM 위상 잔류)                    | 전체 경로(비트 재프로그램) 재실행 — 의도된 방어 |
| 한 스텝에서 BER 10⁻³→0.25 점프           | 버스트 내 프레이밍 슬립                          | 분석기 체크 5가 무효 처리 — 원인 조사          |
