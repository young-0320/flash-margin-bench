# 6. SPI와 클럭 포워딩 — 받아쓰기 시험을 계측기로 만드는 법

- 관련 결정: 계약 R9 (SCLK = clk_core ODDR 포워딩) · 로그 10 ③(0x0B 통일)·④(사전 쓰기 경로) · 로그 11 (SPI 마스터 RTL)

## 쉬운 버전 — 받아쓰기 시험

**SPI는 받아쓰기 시험이고, 시간(클럭)은 마스터가 알려준다.** 플래시 칩은 자기 시계가 없다. 마스터가 SCLK를 한 번 칠 때마다 한 비트씩 주고받고, 클럭이 멈추면 시험도 멈춘다. 문장의 시작과 끝은 CS가 알린다 — CS가 내려가면 받아쓰기 시작, 올라가면 끝. 중요한 건 칩이 **CS 하강 이후 클럭 수를 세는 것만으로** 지금이 몇 번째 비트인지 안다는 점이다. 그래서 프레임당 클럭 수는 정확히 약속돼 있어야 하고, 하나라도 어긋나면 마스터와 칩이 서로 다른 문장을 읽는다.

**빨리 읽으려면 칩에게 준비 시간을 줘야 한다.** 주소를 불러주면 칩이 그 페이지를 내부 배열에서 꺼내올 시간이 필요하다. 0x03은 주소 직후 바로 답을 요구해서 저클럭 전용이고, 0x0B는 주소 뒤에 **더미 8클럭**을 비워 준다 — 낭비가 아니라 고클럭의 입장료다. 측정은 0x0B 하나로 통일했다. 사다리 도중에 명령을 바꾸면 곡선끼리 비교가 안 되기 때문이다.

**핵심: SCLK를 따로 만들지 않고 clk_core 자체를 내보낸다 (클럭 포워딩, R9).** 우리 측정은 "클럭과 살짝 어긋난 시점에 듣기"다(위상 스윕). 그 실험이 성립하려면 칩이 받는 SCLK와 우리가 어긋내는 샘플 클럭이 **정확히 같은 빠르기**여야 한다. SCLK를 2분주로 만들면 데이터 1비트가 우리 클럭 두 주기가 되어, 한 주기만큼 도는 위상 스윕으로는 비트의 절반밖에 못 훑는다.

**그리고 반 박자의 지혜.** 비트가 바뀌는 순간에 읽으면 바뀌다 만 값을 읽는다. 그래서 MOSI·CS는 클럭 경계에서 바꾸고, SCLK는 반주기 밀어 내보내 칩이 **사이클 한가운데서 읽게** 만든다 — 바꾸는 순간과 읽는 순간이 언제나 반 박자 떨어져 있어, 타이밍이 "검사해서 맞추는" 대상이 아니라 **기하로 보장**된다.

**마지막으로, SPI 경로는 둘이다.** 같은 JB 핀에 PS 하드 SPI(저클럭)와 PL SPI 마스터(고클럭)가 비트스트림 교체로 번갈아 앉는다. PS 쪽은 JEDEC 신원 확인과 사전 쓰기+검증 담당 — 느리지만 타이밍 논란이 없는 경로라 "애초에 잘못 쓰인 데이터" 사고를 원천 차단한다. PL 마스터는 오직 위상 스윕 읽기만 한다. 이렇게 갈라 두면 스윕에서 나온 에러가 **전부 읽기 타이밍 하나에 귀속**된다 — 저울로 잴 것과 자로 잴 것을 섞지 않는 실험 통제다.

## 엔지니어링 버전

**버스와 mode 0.** 4선: CS(프레임), SCLK(클럭), MOSI(마스터→칩), MISO(칩→마스터). 프레임 = CS 하강~상승, 칩은 CS 하강 이후 클럭 수로 비트 위치를 계수한다(스모크 TB가 프레임당 SCLK = 40+B를 검사하는 이유). mode 0: SCLK 유휴 low, 칩은 **상승 에지에서 MOSI 샘플, 하강 에지에서 MISO 갱신** — 읽는 순간과 바꾸는 순간의 반주기 분리.

**읽기 명령.**

| 명령               | 오버헤드                 | 최대 클럭 | 용도                                    |
| ------------------ | ------------------------ | --------- | --------------------------------------- |
| 0x03 Read          | cmd 8 + addr 24 = 32클럭 | ~50MHz   | 저클럭 유틸리티 (flash_prep 검증 읽기)  |
| **0x0B Fast Read** | + dummy 8 = **40클럭**   | ~104MHz+ | **측정 경로, 전 사다리 통일 (로그 10 ③)** |

더미 8클럭 = 칩 내부 파이프라인이 주소를 배열에서 꺼내올 시간. 사다리 도중 명령을 바꾸면 더미 유무가 통제 안 된 변수로 곡선 비교에 끼므로 0x0B 단일 경로.

**R9 — 클럭 포워딩.** 계약의 전제 등식 **f_SCLK = f_clk_core = f_clk_sample**. SCLK를 2분주로 만들면 1비트 = clk_core 2주기가 되어 1주기짜리 위상 스윕이 UI의 절반만 덮는다. 구현은 ODDR(패드 전용 DDR 출력 플롭)에 D1=0, D2=en:

- en이 레지스터 출력이라 **글리치 없는 게이팅** (유휴 구간 SCLK 정지 = mode 0 유휴 low)
- SCLK 상승 에지가 **사이클 중앙**: MOSI·CS는 사이클 경계(clk_core 에지) 런치이므로 반주기 셋업/홀드가 구조로 확보 — XDC에서 출력 3핀을 false_path 처리하는 근거 (`flash_spi_ctrl.v` 헤더의 타이밍 유도, `g3_chip_pins.xdc`)

**두 SPI 경로.** 비트스트림이 다르므로 동시에 존재하지 않는다 — 갈아끼우는 것.

| 경로                    | 클럭      | 소유               | 역할                                       |
| ----------------------- | --------- | ------------------ | ------------------------------------------ |
| PS 하드 SPI (EMIO→JB)  | ~2.6MHz  | 소프트웨어 (XSpiPs) | JEDEC ID(G2), 사전 쓰기·검증(flash_prep)  |
| PL SPI 마스터 (RTL→JB) | 25~75MHz | flash 블록         | 계측 — 위상 스윕 읽기 (0x0B 전용)         |

분담의 비대칭이 양방향으로 강제된다: PS SPI는 SCLK가 PS 고정 클럭 트리에서 나와 **위상 제어가 불가능**하므로 마진 측정은 반드시 PL 마스터여야 하고, 역으로 PL 마스터에 쓰기(WEL·프로그램·상태 폴링)를 넣으면 계측과 무관한 RTL 표면만 늘어나므로 읽기 전용으로 좁게 유지한다(로그 10 ④, R7의 정신).
