## 1. 위상 아키텍처 (전체 전제)

**샘플 클럭만 시프트한다.** SPI 마스터(SCLK·CS·MOSI)와 모든 제어 로직은 고정 클럭 `clk_core`. MISO **캡처 경로만** MMCM 동적 위상 시프트 출력 `clk_sample`(clk_core와 동일 주파수, 가변 위상)로 구동.

```
MISO 핀 → [캡처 플롭]      → [비교기·에러 로거·e_i 버퍼] → (CDC) → 핸드셰이크/AXI
           @clk_sample          @clk_sample                          @clk_core
SCLK·CS·MOSI ← SPI 마스터 @clk_core  (위상과 무관, 전 스텝 동일)
└─ 캡처~집계~CDC 전부 flash 블록 소유. 계약 경계의 모든 신호는 clk_core 동기.
```

## 2. core ↔ flash 신호 (전부 clk_core 동기)

| 신호               | 방향 (core 기준) | 폭 | 의미                                                       |
| ------------------ | ---------------- | -- | ---------------------------------------------------------- |
| `meas_start`     | out              | 1  | 1사이클 펄스. 측정 버스트(N회 읽기 × B비트) 개시          |
| `meas_busy`      | in               | 1  | 측정 진행 중 레벨                                          |
| `meas_done`      | in               | 1  | 1사이클 펄스. 이 시점부터 다음 start까지 결과 유효         |
| `meas_timeout`   | in               | 1  | 1사이클 펄스, `meas_done`과 동시 — 이번 done이 워치독 강제 종료(R10) |
| `cfg_err`        | in               | 1  | 1사이클 펄스. start가 R11 위반으로 거부됨(busy·done 발행 없음) |
| `err_bits`       | in               | 32 | 총 비트 에러 수 (오버플로 불가)                            |
| `err_reads`      | in               | 32 | 에러 ≥ 1인 읽기 수 (오버플로 불가)                        |
| `log_rd_addr`    | out              | 12 | e_i 버퍼 읽기 인덱스 (0..N−1)                             |
| `log_rd_data`    | in               | 16 | 인덱스 해당 읽기의 에러 수 e_i. 주소 제시 다음 사이클 유효 |
| `cfg_n_reads`    | out              | 16 | N. start 시점에 flash가 래치                               |
| `cfg_burst_bits` | out              | 16 | B (비트, 8의 배수). start 시점 래치                        |

e_i 버퍼: flash 소유 BRAM 1개 (2,048 × 16b). 읽기 하나가 끝날 때마다 그 읽기의 에러 수를 순서대로 기록.

## 3. AXI-Lite 레지스터 맵 v0 (32비트, 워드 정렬, 베이스 주소는 Vivado 배치 후)

| 오프셋 | 이름           | 접근          | 리셋값      | 내용                                                                                                                                                     |
| ------ | -------------- | ------------- | ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 0x00   | `ID`         | RO            | 0x4D42_0100 | 매직 "MB" + 버전 — 브링업 시 주소 디코드 확인용                                                                                                         |
| 0x04   | `CTRL`       | W(self-clear) | 0           | bit0`MEAS_START` / bit1 `PHASE_INC` / bit2 `PHASE_DEC`. 명령 비트는 한 번에 하나 — 2개 이상이면 실행 없이 `CMD_ERR`                                                                                             |
| 0x08   | `STATUS`     | RO            | —          | bit0`MEAS_BUSY` / bit1 `MEAS_DONE`(sticky) / bit2 `PS_BUSY` / bit3 `MMCM_LOCKED` / bit4 `TIMEOUT`(sticky, R10) / bit5 `CFG_ERR`(sticky, R11) / bit6 `CMD_ERR`(sticky, 리셋으로만 클리어 — 호스트 명령 인코딩 버그 검출) |
| 0x0C   | `N_READS`    | RW            | 100         | [15:0]                                                                                                                                                   |
| 0x10   | `BURST_BITS` | RW            | 2048        | [15:0], 8의 배수                                                                                                                                         |
| 0x14   | `PHASE_POS`  | RO            | 0           | signed 32, 리셋 기준 누적 위상 스텝                                                                                                                      |
| 0x18   | `ERR_BITS`   | RO            | 0           | 32비트                                                                                                                                                   |
| 0x1C   | `ERR_READS`  | RO            | 0           | 32비트                                                                                                                                                   |
| 0x20   | `LOG_ADDR`   | RW            | 0           | e_i 버퍼 인덱스 [11:0]                                                                                                                                   |
| 0x24   | `LOG_DATA`   | RO            | 0           | `LOG_ADDR`가 가리키는 e_i [15:0]                                                                                                                       |

**호스트 스텝 시퀀스** (스윕 루프는 PS가 돈다):
`PHASE_INC` → `PS_BUSY`=0 폴 → `MEAS_START` → `MEAS_DONE`=1 폴 → `ERR_*` 리드 + e_i 버퍼 회수(`LOG_ADDR` 쓰기/`LOG_DATA` 읽기 × N회) → CSV 1행

호스트 실행 환경 = **PS bare-metal C** (Vitis standalone, 2026-07-07 확정) — 레지스터는 포인터 직접 접근(Xil_Out32/In32), OS·드라이버 없음. CSV는 UART로 PC에 전송해 PC측에서 파일화.

## 4. 규약 (전부 지켜야 함)

- **R1** core는 위상 이동 완료(PSDONE) 후에만 `meas_start`를 준다 — flash는 위상의 존재를 모른다.
- **R2** 위상 이동은 `meas_busy`=0 (CS high) 구간에서만.
- **R3** 결과(`err_*`, 버퍼)는 `meas_done`부터 다음 `meas_start`까지 유효. **버퍼 회수는 다음 START 전에 완료할 것** (START가 버퍼를 덮어씀).
- **R4** busy(`MEAS_BUSY` 또는 `PS_BUSY`) 중의 CTRL 명령·start는 무시된다. 호스트는 STATUS 확인 후 명령.
- **R5** `MEAS_DONE`은 sticky, 다음 `MEAS_START`가 자동 클리어. 별도 클리어 쓰기 없음.
- **R6** N ≤ 2,048 (버퍼 깊이), B는 8의 배수. 시작 시점 래치라 측정 도중 레지스터 변경은 진행 중 측정에 무영향.
- **R7** G0 루프백도 동일 인터페이스 — 패턴 출력 핀 → 입력 핀 경로가 flash 자리를 대신하며, 실칩 전환 시 계약 불변.
- **R8 무결성 검사 (필수)**: 호스트는 매 스텝 버퍼 회수 후 ①Σe_i == `ERR_BITS` ②count(e_i>0) == `ERR_READS` ③모든 e_i ≤ B를 확인한다. 하나라도 불일치하면 해당 스윕을 `_invalid` 처리하고 중단 — 카운터·버퍼 경로의 구현 버그를 그 자리에서 잡는 층.
- **R9 SCLK 생성**: SCLK는 별도 분주기 없이 **clk_core를 포워딩(ODDR)** 해 출력한다 — f_SCLK = f_clk_core = f_clk_sample. 이 등식이 깨지면(예: 2분주 SCLK) 위상 스윕이 UI 전체를 커버하지 못한다. 계약의 전제 조건.
- **R10 유한 시간 보장 (워치독)**: flash는 `meas_done`을 **T_max = 2 × N × (B + 64) clk_core 사이클** 안에 반드시 발행한다. 내부 워치독이 한계 초과 시 측정을 강제 종료하고 `STATUS.TIMEOUT`(sticky)을 켠다. TIMEOUT 스텝은 무효. **브링업 중 TIMEOUT은 재시도 금지 — 무조건 원인 조사 대상.**
- **R11 설정 검증 (시끄러운 거부)**: start 시점에 래치값을 검사한다 — 1 ≤ N ≤ 2,048, 8 ≤ B ≤ 65,528, B는 8의 배수. 위반 시 측정을 시작하지 않고(busy·done 발행 없음) `STATUS.CFG_ERR`(sticky)를 켠다. TIMEOUT·CFG_ERR은 다음 유효 START에 자동 클리어. (유효 START = 검증을 통과해 측정이 실제 시작된 START, 구현상 `meas_busy` 상승 에지 — 거부된 START는 이전 플래그를 지우지 못한다)

## 5. 수치 계약

| 항목                      | 값                                                                                                                              |
| ------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| B_BITS (읽기 1회 버스트)  | **2,048비트 = 256B = W25Q 페이지 1개** (0x0B fast read, 오버헤드 40비트/회)                                               |
| N (위상당 읽기 수) 기본값 | **100** (G0 실측 σⱼ로 재확인 예정 — 레지스터로 변경 가능)                                                              |
| 결과 카운터 폭            | **전부 32비트** (설정 16비트 전제에서 구조적 오버플로 불가)                                                               |
| 클럭 플랜 (E, 확정)       | **MMCM: M=9, D=1 → VCO 1,125MHz / O=45 → f = 25MHz** (clk_core = clk_sample = SCLK 공통) — 지터 최선 구성(D=1·정수 M) |
| 위상 스텝 크기 Δφ       | **15.87ps** (= VCO 주기 888.9ps ÷ 56). UI = 40ns, 스윕 = 56×45 = **2,520스텝**, 약 21초(N=100)                    |
| MMCM 입력 클럭            | 보드 125MHz, 핀 **K17** (LVCMOS33)                                                                                        |
| G0 루프백 핀              | 출력 **JE1(V12)** ↔ 입력 **JE2(W16)**, 인접 핀 점퍼 직결. JE는 핀당 200Ω 직렬 저항 내장 — 충돌 보험 별도 저항 불요. 실칩(G2)은 고속 PMOD JB 사용 예정 |
| 측정 패턴 (확정)          | **PRBS-15, 다항식 x¹⁵+x¹⁴+1, 시드 = {1, index[13:0]}** (상위 비트 1 고정 → 시드 0 구조적 불가). index = 루프백: 스텝 내 읽기 번호 0..N−1 (전 스텝 동일 자극), 실칩: 페이지 주소 하위 14비트. 개념: `docs/concepts/5.prbs_pattern.md` |

## 6. 스윕 CSV 스키마

- 파일: 스윕 1회 = 1개, `build/data/sweep_<target>_<YYYYmmddTHHMMSSZ>.csv`, target = `loopback` | `chip01`~`chip10`. 게재분만 `docs/reports/` 수동 승격.
- 열 (행 = 위상 스텝 1개, `phase_step` 0부터 연속, 결측 금지):

| 열                                        | 타입       | 출처                                                |
| ----------------------------------------- | ---------- | --------------------------------------------------- |
| `phase_step`                            | int        | 원시 키 (스윕 시작 기준 스텝 인덱스)                |
| `phase_ps`                              | float      | 파생 = phase_step × dphi_ps                        |
| `n_reads`, `b_bits`                   | int        | 설정값                                              |
| `bit_errors`, `reads_with_error`      | int        | `ERR_BITS`, `ERR_READS` 리드값                  |
| `bit_err_sq_sum`                        | int        | **호스트가 e_i 버퍼에서 파생 계산** (Σe_i²) |
| `f_sclk_hz`, `dphi_ps`                | int, float | 클럭 플랜 (E 확정 전 dphi_ps는 자리표시자)          |
| `target`, `generated_at`, `git_rev` | str        | 런 메타데이터 (전 행 반복)                          |

- **원본 e_i 저장 필수**: 스윕마다 동반 파일 `sweep_<target>_<stamp>_reads.csv` (열: `phase_step, read_idx, err_count`)에 읽기별 에러 수 원본 전량 기록 (~27만 행, 수 MB). 메인 CSV와 같은 basename으로 짝을 맺는다. 메인 CSV는 단독으로 분석 파이프라인을 충족하며, 동반 파일 소실 시 읽기 단위 세부만 상실된다. — e_i를 버리면 3-4에서 버퍼안을 채택한 이유(원시값 보존)가 무효가 되므로 선택이 아니라 필수.
- 파생량(BER 등)은 저장하지 않는다. **무효 런의 정의** — 다음 중 하나라도 발생한 스윕은 파일명에 `_invalid` 접미, 분석 입력에서 제외: ① `TIMEOUT` 발생(R10) ② `CFG_ERR` 발생(R11) ③ 스윕 중 `MMCM_LOCKED`=0 관측 ④ R8 무결성 불일치 ⑤ 스윕 미완주·스텝 결측 ⑥ 스윕 종료 시 `CMD_ERR`=1.
- 소비자: `host/analysis/bathtub_analysis.py` (이 스키마가 입력 계약).
