- 경계 판정 계단(B안) vs 보간(C안) 정면 비교 시뮬레이션: 계단은 N을 늘려도 양자화 바닥(수 ps)에서 멈추고 최악 지터에서 규칙 탈락 → 보간 채택 확정. 다중 문턱값 병행 기록(ⓖ)은 제안 상태
- B_BITS=2,048이 추론값임을 경고하는 `!!` 플래그 추가: 변경 시 FPGA 에러 카운터 폭(N×B, 오버플로 시 곡선 어깨 오염)과 N 권고값이 연쇄 변경됨을 명시
- 생성물 정책 정립 및 CONTRIBUTING 반영: 재현 가능 생성물은 전부 `build/` 하위(gitignore), 보고서·발표 게재분만 사람이 `docs/reports/`로 수동 승격 후 커밋. 스크립트 출력 경로를 build/로 이전, 기존 docs 승격본과 바이트 일치 확인
- 스크립트 보완 3건: ① θ 스윕 추가(`mc_theta_sweep.png`) — θ 클수록 폭 편차 단조 감소 발견(θ=10⁻²가 10⁻³ 대비 약 2.4배 정밀), 채택 θ 재논의 필요 ② 시간 모델을 실제 스윕 범위(UI=클럭 한 주기)와 정합 — 25MHz·N=100 기준 스윕 약 23초로 정정(기존 4.6초는 스팬 불일치로 과소평가) ③ CSV에 생성 시각·git 해시 스탬프 추가
- 다중 문턱값 병행 기록(10⁻²/10⁻³/10⁻⁴) 채택 확정. 에러 카운터 폭(D)·클럭 플랜(E) 결정은 보류
- 작업 로그 규칙 확정: 파일명은 영어 소문자+언더스코어, 문서 내 대제목은 한글
- **채택 θ = 10⁻² 잠정 확정 (G0 실측 후 최종)** — 잠정 근거: 노화 검출 SNR(신호/노이즈)이 비탈 위치 이동·기울기 변화 두 시나리오 모두에서 최대(10⁻³ 대비 1.7~2.2배). "에러 없음" 직관은 병행 기록되는 10⁻⁴ 폭이 담당. **N=100 유지** — 하한은 채택 θ가 아니라 병행 기록 최저 θ(10⁻⁴)의 유효 조건 N×B×θ ≥ ~20이 결정. 스크립트 기본값·docstring·개념 문서 반영, 그림 2장 재생성 및 docs/reports/ 승격(N 그림에 adopted N=100 표시 추가)

- `host/analysis/bathtub_analysis.py` 작성 (G0 준비물 ①): 스윕 CSV → 폭 3종(log 보간, MC와 동일 알고리즘 재사용) + 체크리스트 4항 자동 판정 + 곡선·적합 그림. 합성 데이터 자기 검증 통과 — σⱼ 복원 80.4/79.1ps(정답 80), 폭이 해석해와 0.3ps 이내 일치, 과분산 F=1.04
- 체크리스트 1항 판정 척도 정정: log-BER 선형 → **probit(erfcinv) 선형** — 가우시안 꼬리는 log 축에서 2차 곡선이라 log 선형성 검사는 정상 곡선도 탈락시킴. MC docstring 동기화

## 미결 의사결정

- **θ 최종 확정 게이트 (G0 직후)**: G0는 칩·노화와 무관한 계측기 특성 데이터이므로 이를 근거로 확정해도 사후 선택이 아님. 사전 선언 체크리스트 — ① 실측 비탈의 probit(erfcinv) 직선화가 선형(가우시안 꼬리 부합) ② BER floor < 10⁻⁵(10⁻⁴ 병행 기록 비오염) ③ 실측 σⱼ로 MC 재실행해 N=100 재확인 ④ 에러 버스트성 점검(비트 독립 가정). 4항 통과 시 10⁻² 확정, 이후 변경 금지. 미통과 항목이 있으면 그 항목이 가리키는 방향으로 θ 세트 재설계. 판정 실행기 = `bathtub_analysis.py`
- **스윕 로그 CSV 스키마 잠정안 승인 → `docs/interface/` 반영**: phase_ps, n_reads, b_bits, bit_errors + 버스트성 판정용 **bit_err_sq_sum(읽기별 에러 수 제곱합)**, reads_with_error. 제곱합은 FPGA error logger에 누적기 추가가 필요 — 팀원 A와 합의 필요
- **체크리스트 판정 문턱값 확정**: probit R² ≥ 0.98 / floor < 10⁻⁵ / 과분산 F ≤ 2.0 (현재 잠정치, G0 전 확정)
- **D. FPGA 에러 카운터 폭**: 위상당 최대 에러 = N×B. 권고 24비트(N=100·B=2048 대비 80배 여유). 확정 시 `docs/interface/` 계약에 명문화 — 팀원 A RTL 착수 전 필요
- **E. 클럭 플랜**: ① 샘플/SPI 클럭(UI → 스텝 수·읽기 시간·스윕 시간: 25MHz 22.8초 / 50MHz 5.9초 / 100MHz 1.5초, N=100 기준) ② MMCM VCO 주파수(Δφ = VCO주기/56, 현재 15ps는 자리표시자). G0 코어 RTL 설계와 함께 확정, 확정 시 MC 재실행
- **F. golden model 독립성 절차**: 경계 판정 구현(host/analysis)과 oracle(sim/golden)이 동일 소유자 — 시점 분리 재구현 vs 팀원 A의 교차 검증 벡터 중 택일. G1 전까지
- **B_BITS = 2,048 확정**: 스윕 엔진 버스트 길이 확정 시 일치 확인 (스크립트 `!!` 플래그 참조). 변경 시 카운터 폭·N 권고 연쇄 재계산
