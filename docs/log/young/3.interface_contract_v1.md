## 작업 내용

- 형식 확정: 계약서를 바로 쓰지 않고 **결정 항목별 후보·장단점·추천** 문서로 작성 — 7/8 아침 3인이 참고해 결정하고, 채택안만 남겨 확정안으로 갱신하는 흐름 (초기에 확정문 형태로 썼다가 이 형식으로 재작성)
- 결정 간 의존성 정리: 위상 아키텍처(0)가 1·2·4의 전제. 스윕 주체(2-1)는 어느 쪽이든 핸드셰이크·카운터·CSV 불변 — v0 호스트 주도로 시작해도 자율 FSM은 레지스터 추가로만 도입 가능
- 수치 산정: 스텝당 최대 에러 N×B=204,800(18비트), err_sq_sum 최대 N×B²=419,430,400(29비트), 스텝당 측정 8.4ms·스윕 ~23초(25MHz·N=100) — 호스트 주도 스윕의 폴링 오버헤드 무시 가능 근거
- 정합 확인: `bathtub_analysis.py` 로더 실코드 기준으로 CSV 후보 평가 — `#` 주석 헤더는 DictReader가 깨짐(4-2(b) 단점), `bit_err_sq_sum` 컬럼 생략 시 SKIP 경로 동작(4-3(a) 근거), 메타데이터 행 반복은 몬테카를로 CSV 관례
- B1(SCLK 핀만 시프트) 기각 논거: MOSI·CS ↔ SCLK 상대 타이밍이 위상에 따라 틀어져 명령 오염이 읽기 마진과 뒤섞임 — 계측 교란 변수
- **7/7 저녁 결정 세션: 15건 전항 확정** — 0 = A안(샘플 클럭 시프트) / 계약 1 = 전항 (a) / 계약 2 = 전항 (a) / 3-1 = 카운터 32비트 통일(24비트 추천 철회 — 설정 필드 16비트 전제에서 err_bits·err_reads 구조적 오버플로 불가) / 3-2 = 포화+OVF(원칙으로 유지) / 3-3 = B_BITS 2,048(페이지 1:1 대응) / 계약 4 = 전항 (a). MC 스크립트 `!!` 플래그 해소, 전 계약 문서·README 동기화
- **3-4 = (A) 읽기별 에러 수 버퍼 확정**: 품질 기준(원시값 보존 — 정보 포함 관계 A→제곱합 일방향, 사후 재분석·이상 추적, 포화 개념 소멸)으로 sq_sum 누적기(B) 대신 채택. 반영: 계약 1 신호 교체(err_sq_sum·cnt_ovf → log_rd_addr[11:0]/log_rd_data[15:0], BRAM 2,048×16b flash 소유, 버퍼 로깅 시 N ≤ 2,048), 계약 2 LOG_ADDR/LOG_DATA 창 추가·ERR_SQ_SUM·CNT_OVF 삭제, CSV bit_err_sq_sum 열은 호스트 파생으로 재정의(분석기 불변). 박지민 인계 사양 = "읽기 종료마다 e_i를 RAM에 기록"(곱셈기 불요)

- **7/7 밤 추가 결정 3건 (16~18) + 리뷰 반영**: 16 = 클럭 플랜 C1(MMCM M=9·D=1 → VCO 1,125MHz, f=25MHz, Δφ=15.87ps, 2,520스텝/21초 — D=1·정수 M 지터 최선 구성이 분해능 1ps보다 우선) / 17 = 워치독+TIMEOUT(R10, T_max=2×N×(B+64), "done 유한 시간 보장"을 하드웨어 성질로) / 18 = 설정 위반 시끄러운 거부(R11, CFG_ERR — R6를 하드웨어 집행 규칙화). SCLK=clk_core 포워딩(R9)으로 커버리지 함정 봉쇄, STATUS bit4/5 추가, 무효 런 정의 5항 재작성, e_i 원본 저장 필수 승격(동반 `_reads.csv`), R8 무결성 검사 규약화
- **MC 재실행 (Δφ=15.87ps)**: 규칙 std ≤ 7.9ps — 채택 θ=10⁻²·최악 지터 200ps에서 std 1.91ps로 **N=100 유지 확정**
- fault injection과 R11의 관계 확인: 의도적 에러 주입은 "합법 설정 + 데이터 오염"이라 CFG_ERR과 직교 — 충돌 없음. CFG_ERR 자체의 검증은 N=0을 일부러 쓰는 negative test로 가능(시끄러운 거부라서 테스트 작성 가능)
- 욕조 곡선 산출 스크립트의 계약 v1 업그레이드는 별도 작업 → `log/young/5` 참조
- **7/7 심야 결정 3건 (19~21) — RTL 착수 전 마지막 구멍 봉합**: 19 = **측정 패턴 PRBS-15**(x¹⁵+x¹⁴+1, 시드 {1, index[13:0]}, 루프백 index=스텝 내 읽기 번호·실칩 index=페이지 주소 하위 14비트 — 저전이 패턴은 욕조 벽이 안 생기고 "밖 BER 0.5 포화"가 전 통계의 암묵 전제였음을 명시 사양화, 개념: `concepts/5.prbs_pattern.md`) / 20 = **G0 루프백 핀 JE1(V12)↔JE2(W16)**, MMCM 입력 K17 (JE 내장 200Ω 직렬 저항 = 충돌 보험 공짜, 실칩 G2는 고속 PMOD JB) / 21 = **호스트 = PS bare-metal C**(Vitis standalone, CSV는 UART→PC 파일화)
- 계약 경계 표기를 사람 → 블록/폴더명 기준으로 전환 (roles.md와의 불일치 해소 — 총력전 중 배정 유동, 결과물 기준). 폴더 성격 확정: `fpga/rtl/core/`=계측 제어, `fpga/rtl/flash/`=측정 대상 경로(루프백 프런트엔드 포함)
