# 5. 욕조 곡선 산출 스크립트 계약 v1 업그레이드 (게재 1번 소프트웨어 완성)

- 작성일: 2026-07-07

## 산출물

- `host/analysis/bathtub_analysis.py` — 계약 v1 수준 업그레이드 (새 파일 없이 기존 산출 도구 단일 유지)
- `build/data/selftest_sweep.csv` + `selftest_sweep_reads.csv`, `selftest_sweep_wrap.csv` + `_reads.csv` — 계약 스키마 그대로의 합성 리허설 데이터 (일반 + 경계 걸침)
- `build/plots/bathtub_selftest_sweep.png`, `bathtub_selftest_sweep_wrap.png` — 보고서 게재 수준 그림 (θ 3선 + probit 적합 σⱼ·R² + 채택 θ 폭 화살표 주석)

## 작업 내용

- 착수 전 리뷰에서 **실버그 1건 발견**: 위상 원점이 MMCM 정렬에 따라 임의 → 실측에서 윈도우가 스윕 배열 경계에 걸친 데이터(양끝 깨끗, 가운데 에러 산)가 확률적으로 발생하는데, 기존 코드는 욕조 모양(중앙 깨끗)만 가정해 폭·중심 판정이 깨짐
- 업그레이드 6건: ① 원형 재중심화 — 저 BER 최장 원형 구간을 배열 중앙으로 회전(위상 원점은 임의라 재레이블일 뿐) ② **체크 5 신설** — 인접 스텝 BER 연속성: 한 스텝(≈16ps)에 10⁻³→0.25 점프는 σⱼ<7ps를 요구해 물리적으로 불가능 → 프레이밍(비트 정렬) 슬립 전용 검출기 ③ 동반 `_reads.csv` 자동 인식 + **R8 무결성 검사**(Σe_i·count·e_i≤B) + sq 열 부재 시 원본에서 파생 ④ `_invalid` 파일명 거부·phase_step 연속성 검증(계약 §6 무효 규약의 소비자 측 집행) ⑤ selftest를 계약 12열 + git 스탬프 + 동반 원본 파일로 격상, wrap 케이스 추가 — 하드웨어 없이 "CSV→그림" 전 파이프라인 리허설 ⑥ 그림에 채택 θ=10⁻² 폭 주석
- 검증 (positive): selftest 2케이스 — σⱼ=80ps 복원(79.0~82.5), 폭이 해석해와 0.1ps 정합(θ=10⁻²에서 3,627.8ps), wrap/일반 케이스 폭 ~1ps 일치, 체크리스트 5항 전부 PASS
- 검증 (negative): ① 메인 CSV 7비트 오염 → R8 즉시 검출 ② 프레이밍 절벽 주입 → 체크 5 FAIL(건수·위치 표시) ③ `_invalid` 파일명 → 입력 거부. 3종 전부 정확히 동작
- 게재 2번(JEDEC ID) 확인: 계약 기준 장세은 착수 가능. 단 레지스터 맵 v0에 JEDEC 트랜잭션 창구는 없음 — 캡처는 ILA/스코프 파형 경로(자연스러운 산출물), 레지스터 기반 읽기가 필요해지면 v1 확장

## 미결 의사결정

- 체크리스트 판정 문턱값 3개(probit R² ≥ 0.98 / floor < 10⁻⁵ / F ≤ 2.0) — 잠정치, G0 전 팀 확정
- 실측 CSV 확보 = 게재 1번의 남은 사슬 전부 하드웨어: G0 RTL(core·flash 블록) → 비트스트림 → 호스트 스윕 코드(mmap) → 루프백 점퍼(직렬 저항 ~330Ω) → 스윕
- 실측 그림의 `docs/reports/` 승격 — 보고서 문단 정리 시 수동
