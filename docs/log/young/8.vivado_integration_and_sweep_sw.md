# 8. Vivado 통합 + 스윕 소프트웨어 (G0 루프백 — 비트스트림·XSA·베어메탈·수신기)

- 작성일: 2026-07-07
- 참고: `fpga/scripts/`·`ps/`는 README상 각각 팀원 A·B 소유 — 로그 7과 같은 총력전 배정 유동 조항으로 진행. 소유자 복귀 시 이 로그가 인수인계 문서.

## 산출물

- `fpga/scripts/build_g0_loopback.tcl` — 프로젝트 재생성 → BD 조립 → 비트스트림·XSA → ELF까지 **원샷**(끝에서 build_g0_sweep.py를 체인 호출). stage 인자: `bd`(BD 검증만)/`bit`(비트스트림까지)/`all`(기본). 2024.2 버전 가드 포함
- `fpga/constraints/g0_pins.xdc` — K17(클럭)/V12(JE1)/W16(JE2) 핀 + 125MHz create_clock
- `fpga/constraints/g0_cdc.xdc` — clk_core↔clk_sample 양방향 `set_max_delay -datapath_only 40ns` + 루프백 핀 false path. **구현 전용**(USED_IN_SYNTHESIS=false), 클럭 조회 실패 시 하드 에러가 나게 작성(조용한 무시 차단)
- `fpga/boards/Digilent/zybo-z7-20/` — Digilent 보드파일 벤더링(MIT) — PS7 프리셋(DDR·UART1)의 출처, 빌드 자기완결
- `fpga/rtl/core/core_top.v`·`fpga/rtl/flash/flash_top.v` — BD 모듈 참조용 X_INTERFACE 속성 몇 줄 추가(clk_core↔s_axi 연관·FREQ_HZ·rstn 극성). 로직 무변경
- `ps/src/g0_sweep.c` — 베어메탈 스윕(계약 §3 시퀀스 ×2,520스텝, R8 무결성, §6 무효 ①~④·⑥ 현장 판정, UART 행 프로토콜 출력)
- `ps/scripts/build_g0_sweep.py` — unified Vitis CLI(`vitis -s`)로 XSA→플랫폼→앱 빌드 자동화
- `ps/scripts/program_g0.tcl` — xsct JTAG로 bit+elf 보드 프로그래밍 (rst→fpga→ps7_init→dow→con, **Vivado GUI 불필요**)
- `host/capture/sweep_uart_capture.py` — UART 스트림 → 계약 §6 CSV 짝(메인+`_reads`) 파일화, 메타 3열(target/generated_at/git_rev)·타임스탬프 파일명·`_invalid` 리네임은 PC측 담당 (PS는 시계·git 없음)
- `pyproject.toml`·`uv.lock` — pyserial 의존 추가
- `docs/concepts/6.vivado_integration_and_xdc.md` — 통합·XDC 개념 (일반론+비유)

## 작업 내용

- **방식 결정 (인수인계에서 위임)**: tcl 자동화. Vivado 2024.2가 로컬에 있어 배치 모드로 직접 실행·검증 반복 가능, `fpga/scripts/README.md`의 ".xpr이 아니라 tcl이 소스" 컨벤션과 일치
- **계획 변경 1건: "AXI Package IP" → BD 모듈 참조(Add Module)**. 표준 신호명이라 AXI 자동 추론·Address Editor 주소 배정은 동일하게 되고, 재패키징 단계가 없어 RTL 수정이 다음 빌드에 바로 반영(브링업 반복 마찰 + 재포장 망각 사고 유형 소거). 팀 배포가 필요해지면 그때 패키징(가역적)
- BD 구성: Zynq PS(보드 프리셋: DDR·UART1) + core_0 + flash_0 + SmartConnect + proc_sys_reset(ext_reset=FCLK_RESET0_N, dcm_locked=mmcm_locked, slowest_sync_clk=clk_core) + xlconstant 0→mmcm_rst. ACLK 전부 clk_core(계약 §1), clk125는 PS 무관 외부 핀. **core 베이스 주소 0x43C0_0000 (4K)**
- 검증:
  1. BD validate 통과 (PSU-1~4 critical warning은 Digilent DDR 보정값의 알려진 무해 경고)
  2. 합성→구현→비트스트림: 타이밍 **WNS +29.610 / WHS +0.094**, `g0_wrapper.bit` + `g0_loopback.xsa` 생성. 빌드 스크립트가 타이밍 실패 시 에러로 죽게 게이트
  3. **CDC 제약 확증**: 라우팅된 체크포인트의 `report_exceptions`에 max_dpo=40 양방향 + 핀 false path 등재, 국경 경로(err_reads→s_axi_rdata, b_lat→flash_rx)가 실제 그 예외로 타이밍되는 것 확인 — "제약을 썼다 ≠ 물렸다" 간극을 직접 닫음
  4. `sweep_uart_capture.py`: pty 가짜 UART로 엔드투엔드 테스트 — 정상 스윕(VALID, 열·메타 정합)과 중단 스윕(`_invalid` 리네임이 분석기의 `<stem>`↔`<stem>_reads` 짝 규칙 유지) 둘 다 통과
  5. `g0_sweep.c`: build_g0_sweep.py로 실제 BSP(-Wall -Wextra)에 대고 경고 없이 컴파일·링크 — `build/vitis/g0_sweep/build/g0_sweep.elf` (text 25KB). 원샷 체인(bit→elf)도 클린 빌드로 통째 검증
  6. **실보드 스모크(점퍼 미연결, 7/7 밤)**: program_g0.tcl로 프로그래밍 → UART에 `SWEEP BEGIN` 후 `END valid=0 reason=timeout` — 무입력 상태의 설계된 정답(TB 7절 단선 워치독의 실물 재현). 비트스트림·MMCM lock·주소 디코드·레지스터 맵·R10·무효 판정·UART 전부 생존 확인
- 스윕 C 설계 메모: 스텝 0은 리셋 위상 그대로 측정(INC는 스텝 1부터) — `phase_step`과 하드웨어 `PHASE_POS`가 1:1이 되고 매 스텝 대조(호스트 루프 자체 검사). 폴링에 소프트웨어 안전 한도(R10 T_max 16.9ms의 수백 배) — AXI 고장까지 무한 대기로 안 이어지게. Σe_i²는 u64(최대 4.2e10 > 2³²). phase_ps는 정수 fs 연산으로 출력(xil_printf 부동소수 미지원)
- UART 행 프로토콜: `#G0` 메타 / `M,`메인 행 / `R,`reads 행. 무효 ①~④·⑥은 PS가 그 자리에서 판정해 `END valid=0 reason=...`으로 중단, ⑤(미완주)는 END 미수신으로 PC측이 판정 — 계약 §6 무효 정의 전 항목이 정확히 한 곳씩에서 집행됨
- **커밋 전 자체 검토에서 버그 1건 (sweep_uart_capture, 수정 완료)**: 초기 구현은 `END valid=1` 수신만으로 VALID 저장 — valid=1은 "PS 기준 무결"이지 "PC까지 전부 도착"이 아니다. 수신기를 늦게 띄우거나(BEGIN·앞 행 놓침) 전송 중 행이 유실돼도 모자란 CSV가 VALID로 남는 구멍. ⑤ 중 전송 결측은 PS가 알 수 없어 **PC 수신기만이 집행 가능한 반쪽**인데 그게 빠져 있었음. 수정 = BEGIN에서 steps·n 파싱 → END에서 행수 완전 일치(main=2520, reads=252,000)일 때만 VALID, 불일치는 `_invalid`+결측 위치 출력. pty 테스트에 "PS valid=1 + 행 1개 유실 → INVALID" 케이스 추가로 검증. 같은 검토에서 핀 배정(마스터 XDC 대조)·캡처 플롭 IOB 실배치(`ILOGICE2.IFF`)도 실물 확인 — 이상 없음

## 미결 의사결정

- **계약 7/8 추인 4건 여전히 미확정** (§2 신호 2개 / CMD_ERR / R11 유효 START / §6 무효 ⑥) — 이번 통합·스윕 C 모두 제안안 기준으로 구현됨. 부결 시 g0_sweep.c의 해당 검사도 재작업
- UART 115200 기준 `_reads` 전송이 스윕당 약 6분 (본 측정 21초의 지배 항). G0 1회성이라 수용, 거슬리면 XUartPs로 921600 상향은 코드 몇 줄
- 클래식 xsct 배치 플로우(`platform create`)는 이 설치본에서 Vitis 서버 접속 타임아웃으로 재시도 포함 2회 실패 → **unified Vitis Python CLI로 전환**(build_g0_sweep.py). xsct의 JTAG 명령(connect/fpga/dow)은 hw_server 경로라 별개 — program_g0.tcl이 사용
- 실기 순서(다음 세션): 점퍼 JE1-JE2 연결 → (빌드는 이미 돼 있음; 재빌드 시 `build_g0_loopback.tcl` 원샷) → `sweep_uart_capture.py` 수신 대기 → `xsct ps/scripts/program_g0.tcl`로 bit+elf 프로그래밍 → 스윕 자동 시작 → `bathtub_analysis.py` 첫 곡선. JTAG 부팅 모드(JP5) 확인
