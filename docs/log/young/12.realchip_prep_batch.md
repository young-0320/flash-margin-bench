## 작업 내용

- 니어미스 1건: phase_ps 계산의 `40,000,000fs`(=25MHz UI) 상수가 사다리에서 잔류 — 45/75 ELF의 phase_ps가 틀리게 찍힐 뻔. 생성 사본 검수에서 발견, Δφ가 VCO 고정임을 이용해 클럭 독립형 `step × 10⁶/63 fs`로 원본 수정(25MHz 정수 출력과 완전 동일 — 40×10⁶/2520 ≡ 10⁶/63). 2차 방어선: 어제 넣은 분석기 phase_ps 교차검사(로그 10 ②)가 현장에서도 무효 런으로 잡았을 유형
- 45/75 빌드의 X_INTERFACE FREQ_HZ(25MHz 고정 표기)는 BD에서 실주파수로 교정 — STA는 MMCM 파생 클럭 기준이라 메타데이터 문제이나 IP 구성 오판 방지

## 실칩 현장 절차 (요약 체크리스트)

1. 모듈 배선(로그 9 ①②) → `program_g2.tcl` + 세은 JEDEC 앱 → `EF 40 17` = **승격 판정**
2. `xsct ps/scripts/program_g2.tcl build/vitis_prep/flash_prep/build/flash_prep.elf` → UART `#PREP PASS` 확인 (FAIL이면 스윕 금지)
3. `xsct ps/scripts/program_g3.tcl 25` → `sweep_uart_capture.py --target chip01` → `bathtub_analysis.py`
4. 풀 스윕 에러 0이면 45 → 75로 승격 (결정 ⑤). 벽 보이는 최저 클럭에서 체크리스트 통과 → 채택, 연속 반복 5회
5. 판독표: 단선/무칩 = 스텝 2에서 `line_dead` 자가진단 종료(7/8 추가 — 못 잡는 변종이면 "TIMEOUT 없이 완주 + 대량 에러", 로그 11) / 전 위상 BER 0.5 = PAY_LEAD 재보정(75MHz 임계) / **B는 실칩에서 2,048 고정** (페이지=시드 경계)

## 리뷰 라운드 (wf_16617fb7, 7/8 새벽 자동 재개분) — 발견 10건, 수정 4건

멀티에이전트 리뷰(파인더 16 + 발견별 독립 검증) 결과. CONFIRMED·기계적 건은 수정 완료(스모크 4종 재통과 + 3벌 재빌드), 설계 판단 건은 아래 미결로:

- **[수정]** `flash_prep.c` %lu 4곳 → %u: 2024.2 BSP xil_printf는 SUPPORT_64BIT_PRINT라 %lu가 64비트 va_arg — u32 인자면 출력값·후속 인자 정렬이 전부 깨진다. #PREP FAIL의 페이지/바이트 수가 쓰레기로 찍힐 뻔 (g0_sweep.c는 원래 %u 관례)
- **[수정]** `g3_cdc.xdc` 신설, g0_cdc.xdc 재사용 중단: ① 루프백 전용 pattern_in/out false_path가 G3에서 빌드마다 크리티컬 워닝 2건을 "정상"으로 만들던 것 제거 ② max_delay 40ns(25MHz 1주기)를 13.333ns(75MHz 1주기, 전 단 공통 하한)로 — pay CDC 경로가 1주기를 넘으면 PAY_LEAD 유도 전제가 깨지는데 40ns면 75MHz에서 3주기 경로도 합법이었다 (침묵 붕괴 위험)
- **[수정]** PAY_LEAD 배관: flash_top_spi 파라미터 추가 + build_g3_chip.tcl에 클럭별 표(`array set pay_lead`) — 로그 11·12가 처방한 "75MHz면 PAY_LEAD 1 감소"를 소스 포크 없이 실행 가능하게 (O_DIV와 같은 경로)
- **[수정]** FREQ_HZ 오버라이드 catch→하드 에러: 실패가 경고로 스크롤되면 XSA가 25MHz 메타데이터를 물려받은 채 완주한다. 기존 3벌 로그에서 오버라이드 성공 확인 후 강경화
- **[수정]** u_rx.cap IOB 워닝 소거 (`g3_cdc.xdc`): 실칩 top의 진짜 캡처는 cap_miso(IOB)이고 u_rx.cap은 심 뒤 내부 단 — 무의미한 IOB=TRUE가 빌드마다 크리티컬 워닝(Place 30-73)을 남기던 것을 XDC에서 해제(flash_rx RTL 무수정 유지). 재빌드 후 3벌 크리티컬 워닝 0
- 검증: 스모크 4종(tb_flash_spi/tb_flash/tb_core/tb_g0) 전부 PASS, flash_prep ELF 재빌드, 사다리 3벌 재빌드 타이밍 전부 양수 (13.333ns 제약이 물기 시작해 25MHz WNS 29.7→5.8ns — 의도된 동작)
- **[보험] PAY_LEAD 4/6 비트스트림 2벌 사전 빌드** (25MHz, `build/vivado_g3_25_pl4`·`_pl6`): PAY_LEAD=5는 해석 유도+시뮬 검증뿐 실보드 실증 전 — 어긋나면(전 위상 BER 0.5) 현장 재빌드 5분×반복 대신 `program_g3.tcl 25 pl4|pl6`로 몇 초 교체. build_g3_chip.tcl에 오버라이드 인자(`-tclargs bit <mhz> <k>`), program_g3.tcl에 `pl<k>` 선택 추가. 45/75는 25MHz 실측으로 역산 보정이 정도라 미리 안 굽는다

## 미결 의사결정 (리뷰 발견 중 판단 대기 2건 포함)

- **[결정·구현 7/8]** SPI 무칩 자가진단 — 2층 구조(B안, 사용자 승인): 무효 런 정의는 수정 가능 확인.
  - **1층 (결정론적 죽음 → 스텝 2 조기 종료)**: 위상을 바꿨는데 e_i 벡터가 비트 단위로 동일 + 전 읽기 유에러 = 위상 무반응. 검출은 100%(정의상), 극성 분류는 C가 기대 PRBS 1 개수를 즉석 계산해 대조 — `line_dead_stuck0` / `stuck1` / `unknown`(특수 결정론 고장). 검사를 행 출력 뒤에 두어 중단 시점까지의 데이터 + `#G0 DIAG` 근거 줄이 캡처에 남음. 조기 종료는 정보 손실 0인 경우에만(2,520스텝 = 같은 행 복사본)
  - **2층 (애매한 고장 → 완주 후 무효)**: 저에러 스텝(BER<0.25)이 하나도 없이 완주하면 `no_window` — 간헐 접촉류는 전체 에러 목록을 남기고 _invalid로. 리뷰 잔여 구멍(간헐 고장 valid=1)도 폐쇄
  - 루프백은 단선=TIMEOUT 선행이라 1층에 안 오고, 정상 런은 floor 스텝이 2층을 통과시킴
- **[결정·구현 7/8]** flash_prep 전 범위 준비 (리뷰 #7): N_PAGES 100→**2,048**(R6 상한, 512KB·~1분) — N을 어디로 올려도 미준비 페이지 없음. B>2,048 실칩 무의미(페이지=시드 경계)는 flash_prep.c 헤더·아래 체크리스트에 명기
- flash_prep·g3 ELF는 실보드 미실행 — XSpiPs 경로는 세은 G2 앱이 먼저 밟으므로 프리스케일러·초기화 코드의 실증은 거기서 얻는다
- 로그 10 ⑥(SPI RTL 소유)·통지 잔무는 그대로 — 이 배치로 `ps/src`·`ps/scripts`·`fpga/scripts`·`fpga/constraints`에 파일이 더 늘었다 (통지 묶음 = 로그 8~12)
