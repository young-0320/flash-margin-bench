# 11. SPI 마스터 RTL (0x0B) — 실칩 flash 프런트엔드 구현 + 스모크 통과

- 작성일: 2026-07-08 (심야)
- 배경: 게재 4번(실칩 곡선)의 병목 선행물. 로그 10 대기 ⑥의 권고안("Young 구현 + 박지민 cocotb 채점")을 총력전 연장으로 착수 — 박지민 합류(7/8 오후) 시 이 로그가 인수인계·리뷰 기준.

## 산출물

- `fpga/rtl/flash/flash_spi_ctrl.v` — clk_core 도메인 0x0B fast read 시퀀서. R11 검증·R10 워치독은 flash_ctrl(루프백)과 동일 골격. 읽기 i → 페이지 i (addr=i×256, 로그 10 가정 2). SCLK는 sclk_en만 만들고 ODDR이 포워딩(R9)
- `fpga/rtl/flash/flash_spi_shim.v` — clk_sample 도메인 프레이밍 심: 마스터의 pay(데이터 창 예고, 2FF 동기) 수신 시 합성 프리앰블 1×4를 붙이고 MISO로 먹스. **flash_rx·flash_logbuf를 무수정 재사용**하게 하는 어댑터
- `fpga/rtl/flash/flash_top_spi.v` — 계약 §2 인터페이스 동일(R7)한 실칩 top. 루프백 flash_top과 병존(로그 10 가정 1), core·스윕 C·계약 무수정
- `sim/smoke/tb_flash_spi_smoke.v` — W25Q64 행동 모델(0x0B, mode 0, PRBS 방출 컨벤션 = flash_prbs15) 내장 스모크. **9시나리오 전부 PASS**
- `sim/smoke/unisim_stub.v` — ODDR SAME_EDGE 행동 모델 추가
- 회귀: tb_flash_smoke(루프백)·tb_g0_smoke(통합) PASS 유지. 루프백 TB 실행 명령에 unisim_stub.v 추가 필요해짐 (파일 헤더 갱신)

## 작업 내용 (설계 요점)

- **프레이밍이 유일한 신규 난제**: 루프백은 프리앰블이 페이로드와 같은 배선을 타서 정렬이 지연 무관이지만, SPI 칩은 프리앰블을 못 보낸다. 대신 마스터가 데이터 도착을 예고(pay)하고 심이 로컬 프리앰블을 합성 — 2FF 동기(±1)와 캡처 체인을 타이밍 산식으로 유도해 **PAY_LEAD=5**(pay 상승 = cyc 35)에서 페이로드가 flash_rx 3후보 창 {t0+3,4,5}에 정확히 들어온다 (산식은 flash_spi_ctrl 헤더)
- SCLK = ODDR(D1=0, D2=sclk_en): 상승 에지가 사이클 중앙 → 에지 런치 MOSI·CS에 반주기 셋업/홀드 자동 확보(mode 0, 유휴 low). CS 셋업 2·홀드 3·갭 4사이클, 읽기당 오버헤드 49 < R10 여유
- 주소→시드 정합은 구조적: RX가 기대하는 시드 {1, idx}와 마스터가 보내는 addr=idx×256이 같은 idx에서 파생 — TB에서 암묵 검증(주소 틀리면 에러 0 불가)
- 검증 커버: 위상 스윕(0.5~39.5ns)·tCLQV 스윕(1~38ns) 에러 0, 프레임 SCLK 수=40+B(슬레이브 검사), 단일 비트 주입 정확 계수+R8, R11 3종, R10(rx_done 강제 차단), N=2048 경계

## 현장 판독 주의 (루프백과 다른 고장 서명)

- **무칩/단선 = TIMEOUT이 아니라 "완주 + 대량 에러"** — 합성 프리앰블이 항상 프레이밍을 성립시키기 때문. 라인 0 고정 시 에러 수 = 기대 PRBS의 1 개수 (B=2048이면 읽기당 ~절반). 루프백의 단선=TIMEOUT과 반대이니 브링업 때 혼동 금지
- **클럭 상향 시 PAY_LEAD 재보정 조건**: tCLQV+배선 왕복이 1UI를 넘으면(75MHz에서 ~0.9-1.1UI로 임계) 정렬이 후보 창을 벗어남 — 증상은 전 위상 BER 0.5. PAY_LEAD 1 감소로 해결, 사다리 빌드 3벌 만들 때 클럭별 확인
