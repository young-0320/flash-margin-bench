## 작업 내용

- **게재 목록 확정 (4항목)**: ① 루프백 욕조 곡선 ② 보드-모듈 통신 증거(JEDEC ID 캡처 — 독립 섹션 아닌 실칩 곡선 도입부) ③ 몬테카를로 근거 그림 2장(확보됨) ④ 실칩 곡선 + 연속 반복 5회 에러바
- **cocotb 게재 강등**: 필수 아님 — 마감 시점에 회귀 PASS 존재하면 캡처 1장으로 줍기. G1 작업 자체(팀원 A 트랙, 시뮬레이션 선검증으로 G2 브링업 리스크 감소)는 유지
- **총력전 모드 선언 (7/7 ~ 제출)**: 폴더 소유·역할 분담·PR 규칙 일시 정지, 전원 크리티컬 패스(G0→G2→G3) 투입. 제출 후 복원
- 총력전 중에도 유지하는 것: 인터페이스 합의 4건(core↔flash 핸드셰이크, AXI-Lite 레지스터 맵, 에러 카운터 폭·집계, UART/CSV 스키마)은 구두로 정하는 즉시 `docs/interface/`에 한 줄 기록
- 스트레스 테스트(마모)는 게재 제외 확정 — G5 게이트 위반(자기모순), 시료 비가역 소실, 크리티컬 패스 잠식. 대신 방법론 서술("마모를 안 한 이유")로 대체
- **7/8 담당 배정 (7/7 저녁 개정 — 박지민 오전 인턴 반영)**: 한영웅 = 인터페이스 계약(아침 최우선, 장세은 언블록) → ① 루프백 곡선 → ③ MC 마무리 → ④ 실칩 통합(저녁). 장세은 = 계약 수령 후 ② SPI·JEDEC ID 브링업(①과 병렬). 박지민(오후) = cocotb 사다리: 환경 구축 → Winbond 모델 구동 → JEDEC ID 트랜잭션 시뮬 PASS(2번의 리허설·향후 SPI 채점기)
- **스크립트 개명**: `mc_reads_per_phase.py`는 이름이 불투명한 약어(mc)이고 범위(θ 스윕까지 포함)보다 좁아 `monte_carlo_sweep_params.py`로 개명. 출력 파일명·`bathtub_analysis.py`의 import·docstring 상호 참조·개념 문서(`concepts/1.monte_carlo.md`)·이전 로그(`log/young/1`)의 경로 언급을 전부 동기화
