# 14. 세은 브링업 코드 2벌 리뷰 + 실기 검증

- 작성일: 2026-07-08
- 배경: 실칩의 날(런북 3 트랙 D) 착수 전, 세은이 "보드-모듈 통신 확인" 용도로 작성한 C 코드 2벌을 리뷰·실행 요청받음.

## 산출물

- 리뷰 판정 (본 로그) — **두 버전 모두 PS↔PL(core 블록) AXI-Lite 테스트**로, 게재 2번(보드↔W25Q64 통신 증거 = XSpiPs JEDEC "EF 40 17", 로그 9 ③)과는 **검증 계층이 다름**
- 실기 실행 증거 2건 (스크래치패드 빌드, 리포 밖): 버전 1 = G0 설계 위 6/6 PASS, 버전 2 = 무한 정지

## 작업 내용

- **정적 대조**: 버전 1의 레지스터 맵·STATUS 비트·리셋 기본값·R1~R11 준수를 contract.md §3·§5와 전수 대조 — 전부 일치. TEST6(CMD_ERR 음성)의 전제(다중 비트 = 실행 없이 sticky)도 `core_axil_regs.v` 구현과 일치. 자리표시자 `0x43C00000`은 우연히 G0/G3 실제 배치와 동일 (버전 2의 `0x40000000`은 불일치).
- **버전 1 실행** (G0 XSA 위 빌드, `-Wall -Wextra` 무경고): 6/6 PASS. JE 점퍼가 트랙 C 이후 그대로 붙어 있어 TEST4·5가 실측정 경로(core↔flash 루프백)까지 완주.
- **실행으로만 잡힌 버그**: `xil_printf`의 `%lu`는 인자를 64비트로 소비 → u32 인자와 어긋나 진단 숫자 전부 오염 + 뒤따르는 인자 정렬까지 밀림 (예: `N_READS write=2199023255602`). **PASS/FAIL 판정은 u32 비교라 유효** — 출력만 거짓말. 수정은 `%lu`→`%u` 일괄 치환.
- **버전 2 실행** (원문 그대로): 헤더 + "접근 시도 (주소: 0x40000000)..." 출력 후 정지. 미배치 AXI 주소 리드 = DECERR = data abort — 코드의 [FAIL] 안내 분기는 이 실패 모드에서 **도달 불가능**함을 실증.
- 보드는 버전 2 정지 상태로 종료 — 다음 `program_*.tcl` 실행이 rst -system으로 회복하므로 조치 불요.

## 후속 (같은 날, Young 결정 3건 실행)

결정: ① PS↔PL 스모크는 유지 가치 있음 ② 버전 1을 수정본으로 채택 ③ 게재 2번용 코드 생성.

- **버전 1 수용**: `ps/src/core_smoke.c` (작성 장세은, 헤더에 명시) — 수정은 %lu→%u 전체 치환(flash_prep.c:63에 이미 명문화된 리포 관례: xil_printf는 SUPPORT_64BIT_PRINT)과 자리표시자 주소 확정(0x43C00000, G0·G3 build tcl 공통 배치 확인) 2건뿐, 나머지 세은 원문 그대로. `ps/scripts/build_core_smoke.py`(G0 XSA, 워크스페이스 build/vitis_smoke) + `ps/scripts/program_core_smoke.tcl` 신설. **재실행 검증: 진단 숫자 전부 정상 + 6/6 PASS** (N=100·B=2048, waited 8300us/예산 51688us, 루프백 눈 중심이라 ERR 0).
- **게재 2번용 신규**: `ps/src/flash_jedec.c` (Young 초안, 스펙·검토·캡처 소유 장세은 — 헤더에 명시) — 로그 9 ③ 스펙 그대로 (XSpiPs 폴링, 0x9F, 1초 주기 반복, 사람이 읽는 판정 문구). SPI 초기화·프리스케일러는 flash_prep.c 검증치 재사용. `ps/scripts/build_flash_jedec.py`(워크스페이스 build/vitis_jedec — vitis_prep 보존), 프로그래밍은 기존 `program_g2.tcl <elf>`. **실기 검증: `#G2 JEDEC FF FF FF [FAIL] 응답 없음` 1초 주기 반복** = 칩 미배선 상태의 정상 서명(런북 표 E) — 코드 경로 종단 확인, D0 배선 후 같은 ELF로 EF 40 17 기대.
- **파일명 재고 (Young 지적)**: `core_bringup_test.c`→`core_smoke.c` — "bringup"은 1회성 어감이나 실용도는 사다리(D4) 반복 스모크, "_test" 접미는 리포 밖 관례. `g2_jedec_read.c`→`flash_jedec.c` — `g2_jedec`은 하드웨어 설계(XSA·bit) 이름이라 앱명과 충돌하고 "_read"는 정보량 0, flash_prep(굽기)/flash_jedec(ID 확인)을 G2 인프라 공용 flash_* 가족으로.
- 보드는 flash_jedec(개명 전 동일 바이너리) 반복 실행 상태로 종료 (G2 비트스트림) — 세은이 배선 후 miniterm만 열면 됨. 단 전원 재인가 시엔 `program_g2.tcl <elf>` 재실행 필요.
- **D1 통과 (같은 날 23:42, Young이 현장 배선·실행)**: chip01 모듈을 JB에 암수 점퍼 직결(보호 보드 없이, 전원 2가닥 선검증 절차) 후 flash_jedec 실행 — **`EF 40 17 [OK]` 연속 13회+ 안정 수신** (간헐 증상 없음 = /HOLD 풀업 정상 방증). 화면 캡처 확보 = 게재 2번 증거물. 승격 커밋·소유권 정리(세은 검토)는 미결 참조.
