# 9. G2 JEDEC 브링업 준비 — 하드웨어 절반 완성 

- 작성일: 2026-07-07
- 배경: 게재 목록 2번(보드-모듈 통신 증거, G2 승격 판정 지점)은 장세은 담당. Vivado 절반(BD·비트스트림·핀 배정·프로그래밍)을 Young이 미리 만들어, 세은 몫을 SOP8 배선+베어메탈 C+캡처로 줄인다.

## 산출물

- `fpga/scripts/build_g2_jedec.tcl` — PS SPI0(EMIO)→JB 라우팅만 있는 미니 BD → 비트스트림·XSA (`build/vivado_g2/g2_jedec.xsa`). PL 로직 0
- `fpga/constraints/g2_jedec_pins.xdc` — JB 상단열, Digilent Pmod SPI 표준 배열
- `ps/scripts/program_g2.tcl` — `xsct ps/scripts/program_g2.tcl <elf>` 한 줄 프로그래밍. ps7_init은 XSA에서 자동 추출(앱 워크스페이스 위치 무관), elf 생략 시 비트스트림만

## 세은 할 일 = ③④⑤

① **핀 배정 (확정, 배선 기준표)** — Pmod Interface Spec Type 2 표준:

| JB 핀 | 신호 | FPGA 핀 | W25Q64 (SOP8) |
| ----- | ---- | ------- | ------------- |
| JB1   | CS   | V8      | /CS (1번 핀)  |
| JB2   | MOSI | W8      | DI (5번 핀)   |
| JB3   | MISO | U7      | DO (2번 핀)   |
| JB4   | SCLK | V7      | CLK (6번 핀)  |
| JB5   | GND  | —      | GND (4번 핀)  |
| JB6   | 3V3  | —      | VCC (8번 핀)  |

② **배선 주의 (SOP8 어댑터 직결 시)**: `/WP`(3번)·`/HOLD`(7번)를 **반드시 3V3에 묶을 것** — /HOLD가 플로팅으로 낮게 읽히면 칩이 통신을 일시정지해서 "가끔만 되는" 최악 유형의 증상이 남. 완제품 모듈이면 보통 내장 풀업이라 확인만.

③ **베어메탈 C (`ps/src/`에, 세은 작성)** — XSpiPs 폴링, 스펙:

- 초기화: `XSpiPs_LookupConfig(XPAR_XSPIPS_0_BASEADDR)` — **2024.2 SDT 플로우는 DEVICE_ID가 아니라 BASEADDR을 받는다** (구버전 예제 복붙 시 컴파일 에러 지점)
- 옵션: `XSPIPS_MASTER_OPTION | XSPIPS_FORCE_SSELECT_OPTION`, `XSpiPs_SetSlaveSelect(&spi, 0)`, 프리스케일러 64 (SPI ref 166.67MHz → SCLK ≈ 2.6MHz, JEDEC은 저속 충분)
- 트랜잭션: tx `{0x9F,0,0,0}` 4바이트 `XSpiPs_PolledTransfer` → **rx[1..3] = 0xEF 0x40 0x17** (W25Q64)
- 출력: 1초 주기로 `xil_printf` 반복 (UART 115200) — 캡처 찍기 좋게. 무칩/오배선이면 FF FF FF 또는 00 00 00
- ID 일치/불일치를 사람이 읽게 판정 문구까지 출력하면 캡처 1장이 그대로 게재물

④ **빌드**: 이 머신에서 `ps/scripts/build_g0_sweep.py`를 복사해 XSA 경로만 `build/vivado_g2/g2_jedec.xsa`로 바꿔 `vitis -s`로 빌드 (클래식 xsct 프로젝트 명령은 이 설치본에서 죽음 — 로그 8). 본인 노트북 Vitis를 쓰려면 버전이 2024.2 이상이어야 XSA가 열림.

⑤ **실행**: `xsct ps/scripts/program_g2.tcl <elf경로>` → miniterm으로 UART 확인 → `EF 40 17` 찍힌 화면 캡처 = 게재물 + **4번 승격 판정**.

## 작업 내용

- PS 하드 SPI0을 EMIO로 꺼내 JB에 직결 — PL 로직이 없어 세은이 디버그할 표면이 C 하나로 좁아짐. SS_I=1 고정(0이면 mode fault로 마스터 강등), SCLK_I/MOSI_I=0 고정
- 핀 배정을 Pmod SPI 표준으로 확정 — 4번(실칩 곡선)의 SPI 핀 XDC로 그대로 승계되는 의사결정
- program_g2가 ps7_init을 XSA(zip)에서 직접 추출 — 세은 앱 워크스페이스 구조에 무의존

## 미결 의사결정

- (4번 선행) 읽기 명령 0x03 vs 0x0B, 클럭 상향 판정 기준, 사전 쓰기 절차 — 로그 8 이후 대화에서 식별, 팀 논의 대상
- 프리스케일러 64는 브링업 보수값 — G2는 속도 무관, 4번에서 SPI 마스터 RTL(flash 블록)로 대체되므로 여기서 끝나는 값
