## 확정 5건

① **체크리스트 판정 문턱값 확정** — R² ≥ 0.98 / floor < 10⁻⁵ / F ≤ 2, 잠정치 그대로.
- 근거: 사전등록 원칙 — 곡선을 보기 전에 기준을 동결해야 판정이 유효. floor 기준은 스텝당 N×B = 204,800비트(BER 분해능 4.9×10⁻⁶)와 정합.
- θ = 10⁻²는 여전히 잠정 — G0 곡선에서 체크리스트 4항 통과로 최종 판정 (temp.md 7번, 유일하게 데이터가 정하는 항목).

② **분석기 phase_ps 교차검사 추가** — `phase_ps = phase_step × dphi_ps` 대조, 허용오차 Δφ/2. 불일치 시 무효 런 거부.
- 근거: UART 글자 깨짐의 마지막 미방어 통로 — bit_errors·err_count류는 R8 재검이 상호 대조로 잡지만 phase_ps는 x축에 그대로 쓰이고 있었음. 허용오차 Δφ/2는 "격자 분해능 이상으로 점을 옮기는 깨짐은 전부 검출, C의 fs 절사 오차(<0.002ps)는 무해"의 경계.

③ **실칩 읽기 명령 0x0B(Fast Read) 통일** — 첫 곡선부터. 브링업용 0x03 병용 안(b), 저클럭 0x03 → 상향 시 전환 안(c)은 기각.
- 근거: 계약 §5가 이미 0x0B 기준 / 클럭 사다리 간 곡선 비교에 명령(더미 유무)을 통제 변수로 고정 / 브링업 단순성 역할은 G2(0x9F, PS SPI)가 이미 대체 / 0x03 병행 구현이 오히려 RTL 증가 (0x0B = 0x03 + 더미 카운터).

④ **사전 쓰기는 PS 하드 SPI 별도 ELF("flash_prep")** — G2 인프라(XSpiPs ~2.6MHz, program_g2.tcl) 재사용. 절차: erase → 100페이지 PRBS page program(시드=페이지 주소, 계약 §5) → 같은 저속 경로 read-back 전수 비교 → 통과 시에만 스윕 진행. RTL 쓰기 기능 추가(b), 외부 프로그래머(c)는 기각.
- 근거: 쓰기 오류가 읽기 마진으로 둔갑하는 것을 원천 차단 — 2.6MHz 검증 읽기는 마진 논란 자체가 없음. "중앙 위상에서 검증" 원안은 닭-달걀(중앙 위상은 곡선이 나와야 앎)이라 폐기. flash RTL 읽기 전용 유지(R7), 스윕 C 무수정 원칙 보존. 로그 9 산출물의 수명 연장.

⑤ **클럭 사다리 승격 기준 — 풀 스윕** — "풀 스윕 전 스텝 bit_errors=0 → 다음 클럭 승격(25→45→75MHz), 양벽 각각 BER>θ 스텝 ≥2개 보이는 최저 클럭에서 체크리스트 통과 시 그 곡선 채택". 코어스 스캔 안(8스텝 간격)은 기각.
- 근거: 칩 1개 스프린트에선 차이가 1회성 ~20분뿐인데, 풀 스윕은 `g0_sweep.c` 무수정(코어스는 스트라이드 변형 C 필요)이고 "이 클럭에서 전 스텝 에러 0" 완전 증거 CSV가 보고서 근거물로 남음. 코어스의 놓침(8스텝보다 좁은 에러 구간)은 어차피 R² 통과 못 할 곡선이라 실해는 없으나, 이기는 시나리오가 다칩 캠페인뿐 — 그 규모가 확정되면 재론.
