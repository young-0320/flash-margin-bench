# 7. flash 블록 RTL 구현 (G0 루프백 프런트엔드 — 프레이밍 재정렬 포함)

- 작성일: 2026-07-07
- 참고: `fpga/rtl/flash/`는 README상 팀원 A 소유 — 중간보고서 총력전의 배정 유동 조항(계약 서문, `docs/workflow/1.md` 운영 모드)으로 진행. 소유자 복귀 시 이 로그가 인수인계 문서.

## 산출물

- `fpga/rtl/flash/flash_top.v` — flash 블록 최상위. 계약 §2 포트(추인 대상 2신호 포함) + JE1/JE2 핀 + CDC 결선
- `fpga/rtl/flash/flash_ctrl.v` — @clk_core: R11 검증(시끄러운 거부), TX 버스트 시퀀서(갭32·프리앰블4·PRBS B비트), R10 워치독(T_max=2N(B+64)), run/rx_done 핸드셰이크. busy는 수락된 start 다음 사이클 상승(로그 6 요건 이행)
- `fpga/rtl/flash/flash_rx.v` — @clk_sample: IOB 캡처 사슬, 버스트 검출(0-run 자격 + 프리앰블 에지), **±1 오프셋 3중 병렬 비교기(프레이밍 재정렬)**, e_i·err 누적
- `fpga/rtl/flash/flash_prbs15.v` — PRBS-15 (x¹⁵+x¹⁴+1, 시드 {1,index[13:0]}). **방출 컨벤션 명문화**: dout=lfsr[14], 시드 MSB-first — golden model(파이썬)이 재현할 정의
- `fpga/rtl/flash/flash_logbuf.v` — e_i BRAM 2,048×16, 듀얼 클럭(write@clk_sample/read@clk_core), 읽기 1사이클 지연(계약 §2 규약)
- `fpga/rtl/flash/flash_sync2.v` — 2FF 동기화기(ASYNC_REG)
- `sim/smoke/tb_flash_smoke.v` — flash 단독 TB (위상·지연 스윕 = 재정렬 증명)
- `sim/smoke/tb_g0_smoke.v` — core+flash 통합 TB (§3 호스트 스텝 시퀀스 전체)

## 작업 내용

- **난제의 본질 = "측정기가 자기 장부를 한 칸 밀려 쓰는 사고"**: TX(clk_core)와 RX(clk_sample)는 주파수 동일·위상만 다름 → RX 샘플열은 내용은 같은데 "몇 번째 비트인가"의 정렬이 위상에 따라 통째로 밀림. PRBS는 이웃 비트가 확률 1/2로 다르므로(결정 19) 한 칸 밀린 비교 = 정확히 BER 0.5 — **배선이 멀쩡해도 오답**. 위상 스윕은 15.87ps씩 밀다 비트 경계를 넘는 순간 올바른 정렬이 한 칸 바뀌므로, 고정 정렬이면 스윕 절반이 곡선이 아니라 BER 0.5 벽지가 됨(체크 5가 잡는 "물리적으로 불가능한 절벽"). 곡선 전체를 못 쓰게 만드는데 원인은 물리가 아니라 장부 실수 — 그래서 G0 최대 기술 난제
- **"푼 것"의 정확한 범위**: 정렬 착오가 비트 에러로 둔갑하는 경로를 차단한 것이지 에러 자체를 없앤 게 아님. 전이 경계의 진짜 캡처 실패는 옳은 정렬 비교기가 그대로 셈(그게 재려는 욕조 벽). 즉 **곡선에 남는 것은 전부 물리**가 되도록 계측 아티팩트만 소거
- **해법 = 버스트별 3-오프셋 병렬 비교**: clk_core/clk_sample은 같은 VCO라 주파수 동일(드리프트 0), 위상만 다름. 버스트 파형 [0×32][1×4][PRBS×B]에서 RX가 0-run(≥8) 자격 후 첫 1 샘플을 t0으로 잡고, 페이로드 시작 후보 {t0+3, t0+4, t0+5} 세 개를 PRBS 1개+지연 플롭 2개로 동시 비교, e_i = min(e0,e1,e2). 프리앰블 에지 샘플이 전이 경계에서 0/1 어느 쪽으로 읽혀도 옳은 정렬이 항상 후보 안에 있음 — 고정 정렬의 "스윕 절반 BER 0.5 절벽"(분석기 체크 5 증상)이 구조적으로 불가능
  - 정직한 한계: 포화 영역(진짜 BER→0.5)에서 min-of-3는 0.5를 약간 밑돌 수 있음(B=2048에서 ≈0.49). 통계가 쓰는 벽 적합 영역(BER≤0.25)에서는 옳은 정렬(B·BER)과 틀린 정렬(≈B/2)이 명확히 분리되어 정확
- **CDC 설계(계약 1-1)**: 제어 1비트 2개만 동기화(run→, rx_done←, 각 2FF). 멀티비트 결과는 동기화하지 않음 — rx_done을 올리기 전에 값이 멈추고, R3가 done 이후에만 읽기를 보장하는 준정적 크로싱. e_i 버퍼는 듀얼 클럭 BRAM으로 포트가 도메인별 분리. RX는 리셋 없이 run_s=0 → 전부 초기화(리셋 CDC 소거)
- 버스트 오버헤드 36사이클(갭32+프리앰블4) — 실칩 0x0B 오버헤드 40비트와 같은 규모, R10의 (B+64)×2 여유 안
- 검증 3층:
  1. **tb_flash_smoke** 8절 전부 PASS: ①클린 런 ②**위상 스윕 0.5~39.5ns(14점) 에러 0 = 재정렬 증명** ③경로 지연 0.2~63ns(1UI 초과 포함) ④B=2048 ⑤단일 비트 오염 주입 → e_i/err_bits/err_reads 정확히 1 ⑥R11 거부 3종(N=0·B<8·B%8≠0), busy/done 무발행 ⑦R10 단선 워치독 → done+timeout 동시, 복구 ⑧N=2048 풀 버퍼. R8 무결성 검사를 TB가 매 측정 집행
  2. **tb_g0_smoke** (core+flash 통합) PASS: §3 호스트 스텝 시퀀스 ×3(INC→폴→START→폴→ERR·e_i 회수+R8), R11 사슬(flash 거부→core CFG_ERR sticky→유효 START 클리어), R10 사슬(단선→워치독→core TIMEOUT sticky→복구). 추인 대상 2신호가 실배선으로 처음 검증됨
  3. **Vivado OOC 합성** (xc7z020clg400-1): flash_top 통과, BRAM 추론 확인
- TB 디버그 1건: R11 테스트 실패는 RTL이 아니라 TB가 1사이클 cfg_err 펄스를 폴링으로 놓친 것 — 상시 모니터로 수정 (실물에서는 core sticky가 포집)

## 미결 의사결정

- **위상별 정렬 d의 절벽은 시뮬로 완전 증명 불가** — 시뮬엔 지터·메타스테이빌리티가 없어 전이 경계의 확률적 검출 흔들림은 실측(G0)에서만 나타남. 분석기 체크 5가 최종 심판
- PRBS 방출 컨벤션(시드 MSB-first)의 golden model 반영 — `sim/golden/` 작성 시 flash_prbs15.v 헤더 정의 사용
- 갭 32·프리앰블 4는 구현 상수(계약 미규정) — 실칩 SPI 프레이밍(CS·더미 사이클)으로 교체될 때 RX 검출부만 갈아끼우는 구조
