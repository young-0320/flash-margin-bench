## B. 통지 + ⑥ 확정 

**박지민에게** — 링크 묶음: 로그 8·10·11·12 + `sim/test_plan.md`. 요지 세 줄:

- 총력전 유동 조항(로그 7~8 전례)으로 님 소유 디렉터리(`fpga/scripts/`·`fpga/rtl/flash/`)에 파일 추가됨 — 각 로그가 인수인계 문서
- **⑥ 확정 요청**: SPI 마스터 RTL은 Young이 구현 완료(로그 11) → 님 몫은 cocotb ③(W25Q64 행동 모델)으로 이 RTL을 **채점**하는 것. 승인 기준은 `sim/test_plan.md` F1~F10 (승인 행위는 님 소유)
- 리뷰 요청: `flash_spi_ctrl.v` 헤더의 PAY_LEAD 타이밍 유도가 핵심 검토 지점

**세은에게** — 링크: 로그 9(③④⑤가 님 몫) + 로그 12 체크리스트. 추가된 것: flash_prep(사전 쓰기)은 Young이 만들어 뒀으니 님의 JEDEC 성공 직후 같은 방식으로 실행만 하면 됨 (아래 D2).

## C. 루프백 곡선 (게재 1번) — 지금 바로 가능

준비물: 점퍼선 1개, USB 케이블(JTAG+UART 겸용) 연결, 보드 전원, JP5=JTAG 모드.

```
C1. 점퍼: JE1(V12) ↔ JE2(W16) 직결          # JE 내장 200Ω — 추가 저항 불요
C2. (재빌드 불요 — 비트·ELF 최신 확인만)
    ls build/vivado/g0_loopback.runs/impl_1/g0_wrapper.bit \
       build/vitis/g0_sweep/build/g0_sweep.elf
    # 없거나 의심되면 원샷: vivado -mode batch -source fpga/scripts/build_g0_loopback.tcl
C3. [터미널 1] 캡처를 먼저 켠다 (BEGIN 행부터 받아야 함):
    .venv/bin/python host/capture/sweep_uart_capture.py --target loopback
C4. [터미널 2] 프로그램 + 실행:
    xsct ps/scripts/program_g0.tcl
C5. 완주 대기 (~10분, UART 전송이 지배):
    "#G0 SWEEP END valid=1 reason=complete" 확인
C6. 분석:
    .venv/bin/python host/analysis/bathtub_analysis.py build/data/sweep_loopback_<stamp>.csv
    → 체크리스트 4항 PASS면 θ=10⁻² 최종 확정 (로그 10 ①의 마지막 잔여)
C7. 실측 σⱼ로 몬테카를로 재실행 (게재 3번 잔무 — C6 출력의 "3. sigma_j 추출" left/right 값 사용):
    .venv/bin/python host/analysis/monte_carlo_sweep_params.py --sigmas <left,right>
    → N=100 재확인 (규칙: sigma_width ≤ Δφ/2인 최소 N)
C8. 게재 승격 (수동): build/plots/*.png → docs/reports/ 복사
```

실패 판독: `TIMEOUT` = 점퍼 미연결/접촉 (루프백의 정상 단선 서명) · `phase_pos_mismatch` = ELF만 재로드한 것 — C4 전체 경로로 재실행 · `no_window` = 점퍼는 붙었는데 전 위상 고에러 (배선 길이/간섭 조사).

## D. 실칩 곡선 (모듈 도착 = 트리거)

### D0. 배선 (세은, 로그 9 ①②)

| JB 핀 | 신호 | FPGA 핀 | W25Q64 (SOP8) |
| ----- | ---- | ------- | ------------- |
| JB1   | CS   | V8      | /CS (1)       |
| JB2   | MOSI | W8      | DI (5)        |
| JB3   | MISO | U7      | DO (2)        |
| JB4   | SCLK | V7      | CLK (6)       |
| JB5   | GND  | —      | GND (4)       |
| JB6   | 3V3  | —      | VCC (8)       |

**/WP(3)·/HOLD(7)를 반드시 3V3에 묶을 것** — /HOLD 플로팅이면 "가끔만 되는" 최악 증상 (로그 9 ②).

### D1. G2 JEDEC — 승격 판정 지점 (게재 2번, 세은)

```
# 세은 C 빌드 (로그 9 ③④): ps/scripts/build_flash_prep.py를 복사해
# 파일명만 본인 앱으로 바꾸면 됨 (XSA 경로는 이미 G2 기준)
xsct ps/scripts/program_g2.tcl <세은 elf>
.venv/bin/python -m serial.tools.miniterm /dev/ttyUSB1 115200
→ "EF 40 17" 반복 출력 화면 캡처 = 게재 2번 + 조건부 묶음 승격 확정
```

FF FF FF 또는 00 00 00이면: 무칩/오배선 — D0 표와 /WP·/HOLD 재확인.

### D2. 사전 쓰기 + 무결성 (Young 산출물 실행만, ~1분)

```
xsct ps/scripts/program_g2.tcl build/vitis_prep/flash_prep/build/flash_prep.elf
miniterm으로 확인: #PREP JEDEC → erase(128섹터) → program → 검증
→ "#PREP PASS all 2048 pages verified" 확인 후에만 다음 단계
```

`#PREP FAIL`이면 **스윕 금지** — BP 보호비트/전원/배선 조사. PASS한 데이터는 비휘발 — 지우기 전까지 스윕 몇 번이든 재사용 (반복 5회 포함, 재실행 불요).

### D3. 25MHz 스윕

```
[터미널 1] .venv/bin/python host/capture/sweep_uart_capture.py --target chip01
[터미널 2] xsct ps/scripts/program_g3.tcl 25
```

주의: miniterm과 캡처는 같은 포트라 동시 사용 불가 — 스윕 중엔 캡처만. **재실행은 반드시 program_g3 전체 경로** (ELF만 재로드 = phase_pos_mismatch 거부, 의도된 방어).

### D4. 분석 → 클럭 사다리 (결정 ⑤)

```
.venv/bin/python host/analysis/bathtub_analysis.py build/data/sweep_chip01_<stamp>.csv
```

- **풀 스윕 에러 0** (벽 없음 = 윈도우 > 1UI 증거, 그 자체 보관): → `xsct ps/scripts/program_g3.tcl 45` + 캡처 재실행 → 그래도 0이면 `... 75`
- **양벽에 BER>θ 스텝 각 2개 이상** 보이는 최저 클럭: 체크리스트 4항 통과 → **그 곡선 채택**, D5로
- **전 위상 BER 0.5 = PAY_LEAD 어긋남** (칩 문제 아님):
  - 25MHz: **보험 비트스트림 즉시 교체** (재빌드 불요, 미리 구움) — `xsct ps/scripts/program_g3.tcl 25 pl4` 로 재스윕, 아니면 `pl6`. ELF는 자동으로 기본 것 재사용
  - 45/75MHz: 25MHz에서 맞은 값으로 왕복 지연을 역산해 한 번에 보정 — `build_g3_chip.tcl`의 `array set pay_lead` 수정 → `vivado -mode batch -source fpga/scripts/build_g3_chip.tcl -tclargs all <mhz>` (~5분) → D3 재실행. 급하면 보험식으로 `-tclargs bit <mhz> <k>` 후 `program_g3.tcl <mhz> pl<k>`

### D5. 연속 반복 5회 (채택 클럭, **재장착 금지**)

```
5회 반복: xsct ps/scripts/program_g3.tcl <채택 mhz>  +  캡처 (스탬프가 파일을 자동 구분)
```

### D6. 집계 → 게재 4번

각 CSV를 bathtub_analysis.py에 → 폭(@θ) 5개 → 평균·표준편차 = 에러바. 곡선 1장 + 에러바 표기 = 게재 4번 산출물.

### D7. 원본 보관 (CONTRIBUTING 정책)

실칩 CSV(+`_reads`)는 재현 불가 원본 — `build/data/` → **`data/`로 이동 보관**. 게재 그림만 `docs/reports/`로 승격.

## E. 고장 판독 총표 (현장에서 이것만 보면 됨)

| 관측 (UART/분석기)                            | 뜻                                     | 조치                                            |
| --------------------------------------------- | -------------------------------------- | ----------------------------------------------- |
| `TIMEOUT` (루프백)                          | 점퍼 미연결 — 루프백의 정상 단선 서명 | 점퍼 확인                                       |
| `line_dead_stuck0` (스텝 2 종료)            | MISO 고정 0 — 무칩/단선/전원          | D0 배선·VCC·CS 확인 (#G0 DIAG 수치 동봉)      |
| `line_dead_stuck1` (스텝 2 종료)            | MISO 고정 1 — 단락/풀업 이상          | MISO 배선 확인                                  |
| `line_dead_unknown`                         | 위상 무반응인데 stuck 아님             | MISO가 다른 구동 신호와 단락 의심               |
| `no_window` (완주 후)                       | 저에러 스텝 0개 — 간헐 접촉·전원     | 전체 에러 목록으로 패턴 분석 (이게 디버그 자료) |
| 전 위상 BER ≈ 0.5인데 valid=1 (분석기)       | 정렬이 후보 창 밖                      | 25MHz: `program_g3.tcl 25 pl4`(보험 비트) / 그 외 D4 — RTL 버그 아님 |
| `phase_pos_mismatch` (스텝 0)               | ELF만 재로드                           | program 전체 경로 재실행                        |
| `#PREP FAIL`                                | 쓰기/검증 실패                         | 스윕 금지 — BP비트·전원·배선 조사            |
| JEDEC`FF FF FF`/`00 00 00`                | 무칩/오배선                            | D0 표 + /WP·/HOLD 3V3                          |
| 한 스텝 BER 10⁻³→0.25 점프 (분석기 체크 5) | 프레이밍 슬립                          | 무효 처리 — 원인 조사                          |
