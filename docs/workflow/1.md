- **승격 조건**: 모듈이 D-2(7/8)까지 도착하고 JEDEC ID가 읽히면(G2), 조건부 묶음은 그 즉시 필수로 승격. 미충족 시 루프백 곡선 + 계획으로 보고서를 완성한다.
- **묶음인 이유**: 실칩 곡선이 성공한 순간 재장착 없는 연속 반복 5회는 몇 분~몇 시간짜리다. "곡선 1장 + 에러바"가 "곡선 1장"과 "반복성 표" 두 항목보다 강하므로 분리하지 않는다.
- 병렬 저비용 2개는 하드웨어가 필요 없으므로 무조건 필수와 **동시 진행** (cocotb는 팀원 A 소유).

## 무조건 필수 — 루프백 욕조 곡선 (G0)

칩·DUT 보드 없이 FPGA 단독으로 자기 배선의 욕조 곡선을 실측한다. 외생 변수 0 — 유일하게 확실히 달성 가능한 실측물이자 보고서의 바닥. 최대 기술 리스크(MMCM 위상 제어)를 여기서 소진한다.

0. (선행 입력) 몬테카를로에서 위상당 읽기 횟수 N 확정 — 아래 병렬 저비용 참조
1. MMCM 동적 위상 시프트 제어 RTL + 스윕 FSM 동작 확인
   → verify: 위상 스텝 명령에 대해 시프트 완료(DONE) 응답이 전 구간에서 돌아옴
2. 루프백 경로 구성 (출력 핀 → 입력 핀) + 패턴 생성/비교기 연결
   → verify: 위상 중앙에서 에러 0, 극단 위상에서 에러 발생
3. 위상 전 구간 스윕 → 위상별 에러 카운트 CSV 수집 (UART 경로 포함)
   → verify: CSV에 전 스텝 데이터가 결측 없이 기록됨
4. 욕조 곡선 산출 스크립트 작성 (CSV → 곡선 그림 + 윈도우 경계 판정 + 폭 계산)
   → verify: 곡선이 욕조 모양(중앙 평탄 0, 양끝 상승)이고 폭이 ns 단위로 출력됨
5. golden model(경계 판정 독립 구현)과 폭 판정 교차 검증
   → verify: 두 구현의 윈도우 폭 판정 일치

**산출물**: 루프백 욕조 곡선 그림 1장 + 윈도우 폭 수치

## 조건부 필수 — 실칩 곡선 + 최소 반복 묶음 (G2 → G3 → G4 최소판)

설득력 1위("프로젝트 전체 주장의 증명")지만 달성 확실성이 외생 변수 세 개에 걸려 있어 승격 조건을 단다: ① 모듈 배송 시점, ② 브링업 성공, ③ 저클럭에서는 윈도우가 스윕 범위(클럭 1주기)보다 넓어 에러가 안 보일 수 있음 — 클럭을 올려가며 에러가 보이는 지점을 찾아야 한다.

첫 곡선은 SOP8 ZIF 어댑터 직결 + 저클럭 허용 (전용 보드 불요).

1. 실칩 저클럭 브링업: JEDEC ID 읽기 (G2) — **승격 판정 지점**
   → verify: 0xEF 40 17 (W25Q64) 수신
2. 쓰기 → 읽기 → 검증 기본 트랜잭션 확인
   → verify: 알려진 패턴이 오류 없이 왕복
3. 루프백의 스윕 절차를 실칩 읽기 경로에 적용, CSV 수집
   → verify: 위상 극단에서 에러가 실제로 발생 (안 나면 클럭을 올려 재시도)
4. 동일 스크립트로 곡선 산출
   → verify: 실칩 욕조 곡선 1장 + 윈도우 폭 (ns)
5. 같은 칩 연속 반복 측정 5회 (재장착 없이) — 곡선에 거의 공짜로 붙는 부속
   → verify: 윈도우 폭의 평균·표준편차 → 곡선에 에러바 표기

**산출물**: 실칩 욕조 곡선 그림 1장 (에러바 포함) + 윈도우 폭 평균·편차

## 병렬 저비용 — 몬테카를로: 위상당 읽기 횟수 N의 근거

하드웨어 불필요. 단순 게재물이 아니라 **G0의 선행 입력**이다 — 스윕을 돌리려면 N을 정해야 하므로 사실상 첫 작업.

1. 에러율 p인 위상에서 N회 읽기 시 경계 판정 오차의 시뮬레이션 (Python)
   → verify: N에 따른 윈도우 폭 추정 분산 곡선 출력
2. 채택한 N 값의 근거 문장 정리 (보고서 게재용)
   → verify: "N = ___인 이유" 한 단락 + 그림 1장

## 병렬 저비용 — cocotb 검증 증거 (G1)

하드웨어 불필요. 팀원 A 소유 — 무조건 필수와 동시 진행.

1. cocotb + Winbond 공식 행동 모델 환경에서 SPI 트랜잭션 회귀 실행
   → verify: 회귀 전체 PASS
2. PASS 로그·커버 항목 요약을 보고서 게재 형태로 정리
   → verify: 시뮬레이션 파형 또는 PASS 요약 캡처 확보

## 순수 선택 — 재장착 포함 풀 반복성 통계 (G4 완전판)

조건부 묶음 완료 후, 시간이 남을 때만.

1. 재장착(ZIF 탈착) 포함 반복 측정 N회
   → verify: 재장착 편차가 연속 측정 편차와 분리되어 보고됨

**산출물**: 반복 측정 산점도 또는 편차 표 — "측정이 노이즈가 아니라 신호"의 완전한 증거
