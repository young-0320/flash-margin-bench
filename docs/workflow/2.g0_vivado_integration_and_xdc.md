# 2. G0 Vivado 통합 및 XDC 제약 작성 워크플로

- **작성일**: 2026-07-07
- **목표**: G0 루프백 RTL(core·flash)을 Vivado에서 합성·구현 가능한 상태로 통합하고, 위상 스윕이 타이밍 도구에 의해 조용히 깨지지 않도록 제약을 건다
- **선행 조건**: G0 core/flash RTL 완성 (계약 v1 인터페이스 준수, `docs/interface/contract.md`)
- **산출물**: `fpga/scripts/build_g0_loopback.tcl`, `fpga/constraints/g0_*.xdc`, 비트스트림
- **근거 개념**: `docs/concepts/3.clock_domains_and_cdc.md` (2FF 동기화기 + 준정적 값 — 5번 단계의 예외 선언이 성립하는 이유)

## 1. RTL을 블록 디자인에 올리기

브링업 단계는 RTL을 자주 고치므로 **모듈 참조(Add Module)**를 쓴다 — IP 패키징은 수정마다 재포장이 필요해 마찰이 크다. 신호 이름을 표준(`s_axi_awaddr` 등)으로 맞추면 AXI 버스가 자동 인식된다.
→ verify: 블록 디자인에서 core·flash 모듈이 연결 완료로 표시되고, 재검증(validate design) 경고 0건

## 2. AXI 주소 배정

Address Editor에서 core 레지스터 파일에 주소를 배정한다 (계약: `0x43C0_0000`).
→ verify: Address Editor에 core가 해당 주소로 나타나고, 생성된 소프트웨어 헤더에 그 값이 반영됨

## 3. 물리 제약 — 핀 배정

계약 20의 핀 배정을 XDC에 못 박는다: MMCM 입력 클럭 K17, 루프백 패턴 출력/입력 V12(JE1)/W16(JE2).
→ verify: `report_io` 등에서 미배정(unassigned) 핀 경고 0건

## 4. 타이밍 제약 — 클럭 선언

`create_clock`으로 clk_core 주기를 선언한다 (계약 클럭 플랜: 25MHz → 40ns).
→ verify: STA에서 일반 플롭→플롭 경로 타이밍 위반(setup/hold) 0건 (CDC 국경 경로는 5번에서 별도 처리하므로 이 단계 검사에서 제외)

## 5. CDC 예외 선언 — 가장 사고 나기 쉬운 단계

clk_core↔clk_sample 국경은 위상이 계속 움직이므로, 도구가 "빌드 시점 위상(0) 고정" 가정으로 계산한 타이밍을 그대로 적용하지 않도록 명시적으로 예외를 건다. 우리 국경은 2FF 동기화기 + 준정적 값(`concepts/3`)으로 이미 주기 정렬에 의존하지 않게 설계돼 있으므로 이 예외를 선언할 자격이 있다:

- `set_max_delay -datapath_only 40ns` — 국경 경로: 위상 정렬 검사는 생략하고, 배선 지연만 한 주기 안으로 제한
- `set_false_path` — 루프백 핀 경로: 핀→캡처 플롭 지연 자체가 측정 대상이므로, 도구가 이걸 "고쳐주면" 계측기를 훼손하는 셈이라 검사 자체를 뺀다

빠뜨리면 생기는 사고 두 갈래: ① 도구가 위상 0 기준 타이밍을 강제 → 위상이 이동하는 순간부터 그 계산은 무효인데 **경고 없이** 무효가 됨(실측이 조용히 틀어짐) ② 존재하지 않는 요구를 만족시키려 배선을 뒤틀어 정작 중요한 경로가 나빠짐.
→ verify: 6번의 `report_exceptions` 확인 전까지는 미완료로 간주 (선언 ≠ 적용)

## 6. 제약이 실제로 물렸는지 확증

XDC는 경로를 이름 패턴으로 찾으므로, RTL 계층 이름이 바뀌면 기본 동작은 **조용한 무시**(경고 한 줄)다. 두 겹으로 방어한다:

1. CDC XDC의 클럭 조회를 실패 시 하드 에러가 나는 형태로 작성 (이름 불일치 = 빌드 실패 — 조용한 무시 금지, R11과 같은 철학)
2. 빌드 후 라우팅된 결과에서 `report_exceptions`로 5번 예외가 실제 등재됐는지 확인

→ verify: `report_exceptions` 출력에 5번에서 선언한 국경 경로가 정확한 경로명으로 등재됨. 클럭 이름을 일부러 틀리게 넣어 빌드가 실제로 죽는지 1회 확인(네거티브 테스트)

## 완료 기준

1~6 전부 verify 통과 + 비트스트림 생성 완료 → G0 루프백 실측(워크플로 1번 무조건 필수 항목) 착수 가능.
