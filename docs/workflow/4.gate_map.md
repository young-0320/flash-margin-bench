## G0 — 루프백 곡선

```
빌드(원샷):  vivado -mode batch -source fpga/scripts/build_g0_loopback.tcl
             → build/vivado/g0_loopback.{runs/impl_1/g0_wrapper.bit, xsa}
             → build/vitis/g0_sweep/build/g0_sweep.elf  (체인 호출)
실행:        [1] .venv/bin/python host/capture/sweep_uart_capture.py --target loopback
             [2] xsct ps/scripts/program_g0.tcl          # 원격: 인자 tcp:<호스트>:3121
기대:        "#G0 SWEEP BEGIN" → ~10분 → "#G0 SWEEP END valid=1 reason=complete"
산출:        build/data/sweep_loopback_<stamp>.csv (+_reads)
분석:        .venv/bin/python host/analysis/bathtub_analysis.py <csv>
스모크:      vitis -s ps/scripts/build_core_smoke.py → xsct ps/scripts/program_core_smoke.tcl
             (core 레지스터 계층 6종 PASS/FAIL — G0·G3 비트스트림 공용, 주소 0x43C00000)
```

## G2 — JEDEC 브링업 + 사전 쓰기

```
빌드(설계):  vivado -mode batch -source fpga/scripts/build_g2_jedec.tcl → build/vivado_g2/
빌드(앱):    vitis -s ps/scripts/build_flash_jedec.py → build/vitis_jedec/flash_jedec/build/flash_jedec.elf
             vitis -s ps/scripts/build_flash_prep.py  → build/vitis_prep/flash_prep/build/flash_prep.elf
통신 확인:   xsct ps/scripts/program_g2.tcl build/vitis_jedec/flash_jedec/build/flash_jedec.elf
             miniterm → "#G2 JEDEC EF 40 17 [OK]" 1초 반복  = 게재 2번 캡처
사전 쓰기:   xsct ps/scripts/program_g2.tcl build/vitis_prep/flash_prep/build/flash_prep.elf
             miniterm → "#PREP PASS all 2048 pages verified"  (약 1분, 중간 침묵 정상)
             ※ PASS 없이 G3 스윕 금지. PASS 데이터는 비휘발 — 반복 5회 재사용.
```

## G3·G4 — 실칩 곡선과 반복

```
빌드:        vivado -mode batch -source fpga/scripts/build_g3_chip.tcl -tclargs all <25|45|75>
             → build/vivado_g3_<mhz>/ + build/vitis_g3_<mhz>/g3_sweep/build/g3_sweep.elf
             (보험 비트만: -tclargs bit <mhz> <k> → build/vivado_g3_<mhz>_pl<k>/)
실행:        [1] .venv/bin/python host/capture/sweep_uart_capture.py --target chip01
             [2] xsct ps/scripts/program_g3.tcl <mhz> [pl4|pl6]
기대:        "SWEEP END valid=1 reason=complete" → build/data/sweep_chip01_<stamp>.csv
분석:        bathtub_analysis.py <csv> → 양벽 곡선이면 채택, 전구간 0이면 클럭 상향(사다리)
G4:          채택 클럭으로 5회 반복 (재장착 금지) → 폭 5개 → 평균±표준편차 에러바
보관:        실칩 CSV 원본은 build/data/ → data/ 이동 (D7), 게재 그림만 docs/results/
```

## 공통 규칙·함정 (원전: 런북 3 표 E)

- **수신(캡처/miniterm) 먼저, 송신(xsct) 나중** — 첫 줄부터 받아야 한다
- miniterm과 캡처는 같은 포트(/dev/ttyUSB1) — 동시 사용 불가
- 재실행은 반드시 `program_*.tcl` 전체 경로 (ELF만 재로드 = `phase_pos_mismatch` 의도된 거부)
- 사전 쓰기 없이 스윕 = 전 위상 BER≈0.51, e_i가 PRBS 0비트 수와 일치 (7/8 실사례, 로그 14)
- 전 위상 BER≈0.5 + valid=1 = PAY_LEAD 어긋남 → `program_g3.tcl 25 pl4` (칩 문제 아님)
