
# flash-margin-bench

**FPGA 기반 플래시 메모리 마진 계측 및 노화 진단 플랫폼**

> 불량인지 아닌지를 판정하는 것이 아니라, **여유가 얼마나 남았는지를 측정한다.**

---

## 1. 한 문장 요약

같은 모델의 플래시 메모리라도 개체마다, 온도마다, 사용량마다 "여유(마진)"가 다르다.
우리는 FPGA 보드로 그 여유를 숫자로 측정하고, 여유가 어떻게 줄어드는지를 추적하는 플랫폼을 만든다.

**범위 구조 — Base와 확장**: Base는 SPI NOR(W25Q64 ×10, 확보 완료)로, 측정 방법론 실증부터 노화 추적·수명 역산까지 **NOR 하나로 완결**한다. SPI NAND(W25N)는 확장으로, 실제 비트 에러·Fail Map·ECC가 등장하는 일반화 단계다. 조달·일정 충족 시 착수하며, base의 성립은 확장에 의존하지 않는다.

---

## 2. 배경: 왜 이 프로젝트인가

### 2.1 원래 계획과 그 한계

처음 계획은 "FPGA 기반 SPI Flash Mini-ATE"였다. FPGA로 패턴 생성기, 비교기, ECC, Fail Map, PASS/FAIL 분류기를 만들어 플래시 메모리를 테스트하는 소형 검사 장비다.

그러나

- ATE(자동 검사 장비)는 이미 산업에 존재하는 장비다.
- "기존 장비를 작게 만들었다"는 것은 잘 만든 **구현**이지, 창의적인 **문제 정의**가 아니다.

그래서 장비를 다시 만드는 것이 아니라, **질문 자체를 바꾸기로 했다.**

### 2.2 질문을 바꾸다

|      | 기존 질문                     | 우리의 질문                              |
| ---- | ----------------------------- | ---------------------------------------- |
| 관점 | 이 칩은 정상인가, 불량인가?   | 이 칩은**여유가 얼마나 남았는가?** |
| 결과 | PASS 또는 FAIL (1비트짜리 답) | 여유의 크기와 그 변화 곡선 (연속적인 답) |
| 비유 | 시험의 합격/불합격 발표       | 남은 체력을 재는 건강검진                |

PASS, FAIL은 "고장 났는가"만 본다. 하지만 정말 알고 싶은 것은 "앞으로 얼마나 더 사용할 수 있는가"다. 우리는 메모리 칩에 대해 후자의 질문을 던진다.

---

## 3. 핵심 문제: 정상인 칩에서 무엇을 측정하는가

### 3.1 처음 부딪힌 벽

플래시 메모리는 생각보다 튼튼하다. 데이터시트가 보장하는 범위(온도 85°C, 클럭 104MHz) 안에서는 온도를 좀 올리고 클럭을 좀 바꿔도 **에러가 나지 않는다.** 즉, 조건을 바꿔가며 PASS/FAIL만 기록하면 결과표는 전부 PASS로 채워지고, 실험은 아무것도 보여주지 못한다.

"에러가 나야 실험이 성립한다"는 전제 자체가 함정이었다.

### 3.2 돌파구: 에러 없이도 여유는 측정된다

칩이 데이터를 내보낼 때, 그 데이터가 "올바르게 읽히는 시간 구간"이 존재한다. 이 구간을 **유효 윈도우**라고 부르자.

- 평소에는 윈도우 한가운데에서 읽으니 항상 성공한다. 그래서 여유가 얼마인지 보이지 않는다.
- 그런데 FPGA는 읽는 타이밍을 **수십 피코초(1조분의 1초 단위) 간격으로 아주 조금씩 옮길 수 있다.**
- 타이밍을 앞뒤로 밀어가며 "어디서부터 틀리기 시작하는가"를 찾으면, 윈도우의 폭이 **나노초 단위 숫자로** 측정된다.

이것이 이 프로젝트의 핵심 도구다. **칩이 고장 나기를 기다리는 게 아니라, 고장 나는 지점까지 측정 장비 쪽이 다가가서 여유를 직접 잰다.** 그래서 모든 칸이 PASS인 조건에서도 실험이 성립한다.

### 3.3 여유는 실제로 움직인다

이 윈도우 폭은 고정된 값이 아니다.

- **온도가 오르면** 칩 내부 회로가 느려져 윈도우가 좁아진다.
- **전원 전압이 낮아지면** 역시 좁아진다.
- **칩이 낡으면(쓰기/지우기를 많이 반복하면)** 여유가 줄어든다 — 단, 어느 지표에 나타나는지는 칩 종류에 따라 다르다(아래).

즉 "온도·전압·노화에 따라 여유가 어떻게 줄어드는가"라는 곡선이 우리의 데이터다. PASS/FAIL로는 보이지 않던 것이 숫자로 보인다.

**노화 지표에 대한 정직한 전제**:

- **Base(NOR)**: W25Q는 셀 어레이를 직접 읽는 구조라 노화가 타이밍 윈도우에 반영될 *여지*가 있다 — 단 이것은 보장이 아니라 **base 실험이 검증하는 가설**이다. 그래서 base에서도 노화 지표를 이중화한다: ① 유효 윈도우 폭(가설 검증 대상), ② 소거·쓰기 소요 시간(교과서적으로 확립된 노화 지표 — 윈도우가 마모에 반응하지 않더라도 교정 곡선과 수명 역산은 이 지표로 성립한다).
- **확장(NAND)**: W25N은 셀 → 페이지 버퍼 → 핀의 2단계 구조라, 셀이 마모돼도 버퍼에서 나오는 출력 타이밍은 거의 그대로일 수 있다. 따라서 NAND에서는 위 두 지표에 **위상과 무관한 비트 에러 플로어**(셀 마진의 대리 지표)를 더한 3종 체계로 확장한다.

"윈도우 폭 = 만능 노화 센서"라고 주장하지 않는 것이 이 프로젝트의 방어선이다.

## 4. 무엇을 만들고 무엇을 측정하는가

### 4.1 세 가지 측정

**① 타이밍 여유 측정 (핵심)**
읽기 타이밍을 미세하게 스윕하여 유효 윈도우 폭을 측정한다. 온도와 전압을 바꿔가며 이 폭이 어떻게 변하는지 기록한다.

**② 동작 경계 지도 (shmoo plot, "슈무 플롯")**
전압을 낮추고 클럭을 스펙 한계 너머까지 올리면 실제로 실패하는 지점이 나타난다. 전압-클럭-온도 축에서 "동작하는 영역"과 "실패하는 영역"의 경계를 지도처럼 그린다. 이런 그림을 반도체 업계에서는 슈무 플롯이라 부르며, 우리는 이 측정을 **자동화된 플랫폼**으로 만든다.

**③ 노화 추적 (base: NOR 종단 마모)**

- 같은 칩을 0 → 수만 → 인듀런스 한계(10만 회)까지 마모시키며, 체크포인트마다 윈도우 폭과 소거·쓰기 소요 시간을 측정한다. 칩이 낡을수록 지우기에 걸리는 시간이 길어지는 성질이 있고, FPGA는 이 시간을 10나노초 해상도로 잴 수 있으므로 **동작 시간의 변화 자체를 칩의 나이를 읽는 센서로 쓴다.**
- 마모량-지표 교정 곡선이 확보되면, 이력을 모르는 칩의 측정값 1회로 사용량을 역추정하고 블라인드 실험으로 검증한다.

**③′ 실에러 측정 (확장: NAND)**

- NAND 플래시는 쓰기/지우기를 수만 번 반복하거나 같은 곳을 수십만 번 읽으면 **실제 비트 에러**가 생긴다. 이때 에러가 몇 개, 어느 주소에서 생기는지 지도(Fail Map)로 기록하고 ECC(오류 정정 코드)로 몇 개까지 복구되는지 확인한다.
  - **필수 전제**: W25N은 on-die ECC가 기본 활성이라, 켜둔 채 읽으면 마모로 인한 raw 에러가 전부 정정되어 가려진다. 노화 측정 시 ECC-E 비트로 비활성화하거나 ECC 상태 레지스터를 병행 로깅하는 절차를 프로토콜에 명문화한다.

### 4.2 측정 지표 (전부 단위가 있는 물리량)

- 유효 윈도우 폭 (ns) — 온도별, 전압별
- 에러 없이 동작하는 최대 클럭 (MHz) — 조건별
- 실제 비트 에러 개수와 위치 (확장: NAND, 마모 후)
- ECC로 정정된 개수 / 정정 불가 개수 (확장: NAND)
- 지우기·쓰기 소요 시간 (µs) — 마모 정도에 따른 변화

> 원칙: 임의의 가중치로 합성한 "점수"는 만들지 않는다. 근거를 물었을 때 방어할 수 없는 숫자는 신뢰를 깎는다. 측정값은 항상 단위가 있는 원시 물리량으로 보고한다.

## 5. 시스템 구성

```
[PC / 대시보드]
  │
  │  UART/USB — CSV 저장, 그래프
  │
┌─┴─[ Zybo Z7-20 ]───────────────────────
│
│  PS (프로세서)
│    - 실험 시나리오 자동화
│    - 온도 폐루프 제어 (PID)
│    - 결과 수집
│
│  PL (FPGA 로직)
│    - SPI 컨트롤러 (클럭 가변)
│    - 타이밍 위상 스윕 엔진 (핵심)
│    - 패턴 생성기 / 비교기
│    - ECC 인코더/디코더 (확장: NAND 단계)
│    - 동작 시간 측정기 (10ns 해상도)
│    - Fault Injection (자기 검증용*)
│
└─┬──────────────────────────────────────
  │
  │  본실험: 짧은 배선의 전용 보드 (브레드보드 금지)
  │  단, 첫 욕조 곡선(G0~G3)은 SOP8 ZIF 어댑터 직결 + 저클럭으로 허용
  │
┌─┴─[ DUT 보드 ]─────────────────────────
│
│  - SPI NOR (W25Q64 ×10 확보) — base: 마진 측정·노화 추적 대상
│  - SPI NAND (W25N) — 확장: 실에러·Fail Map·ECC 대상 (조달 시)
│  - 가변 전원 회로 — 전압 축
│  - 레벨 시프터 — 저전압 스윕의 필수 전제
│      (FPGA는 3.3V 출력 고정, DUT 입력 절대최대정격은 VCC+0.4V.
│       없이 전압을 내리면 과전압 스트레스가 실험 교란 변수가 된다)
│  - 히터 + 온도 센서 — 온도 축 (PS가 제어)
│  - 보호 회로
│
└────────────────────────────────────────
```

\* Fault Injection은 "에러를 흉내 내는" 용도가 아니라, **우리가 만든 비교기와 ECC가 에러를 정확히 잡아내는지 스스로 검증하는 용도**다. 측정 장비를 만들 때는 장비 자체의 신뢰성부터 증명해야 하기 때문이다.

같은 이유로 검증에는 설계자 자신과 독립적인 기준(oracle) 3종을 둔다:
**Winbond 공식 행동 모델**(프로토콜 정답, cocotb 시뮬레이션) · **golden model**(경계 판정 알고리즘의 Python 독립 구현) · **루프백 실측**(칩 없이 FPGA 단독으로 자기 배선의 욕조 곡선을 찍어 메커니즘을 물리 검증).

### 왜 FPGA인가 (반드시 받을 질문)

단순히 플래시를 읽고 쓰는 것은 몇천 원짜리 마이크로컨트롤러로도 된다. FPGA가 필요한 이유는 명확히 세 가지다.

1. 읽기 타이밍을 **피코초 단위로 조절**하는 위상 스윕 — MCU로는 불가능
2. 동작 시간을 **10ns 해상도로 계측** — MCU로는 불가능
3. 스펙 한계 너머까지의 **고속 클럭 생성과 실시간 비교** — MCU로는 불가능

즉 FPGA는 "빠른 SPI 통신기"가 아니라 **정밀 계측기**로 쓰인다.

## 6. 이 프로젝트가 하지 않는 것 (정직한 범위 설정)

- 산업용 ATE를 대체하지 않는다. 웨이퍼 검사, 셀 내부 전압 측정 같은 제조사 수준 테스트는 범위 밖이다.
- 상온~60°C 범위의 온도만으로 칩에 에러를 일으키겠다고 주장하지 않는다. (스펙 안쪽이라 에러가 안 나는 것이 정상이며, 그래서 여유 측정 방식을 택했다.)
- 방열판·팬 비교 같은 실험은 하지 않는다. 플래시는 자체 발열이 수십 mW 수준이라 방열 구조의 효과가 물리적으로 측정 불가능하기 때문이다. 온도는 히터로 **제어하는 독립 변수**이지, 완화 대상이 아니다.

범위 밖의 것을 명시하는 이유: 측정을 다루는 프로젝트에서 가장 중요한 자산은 **주장과 데이터가 일치한다는 신뢰**다.

## 7. 실험 계획

| 축   | 값                                                                                              |
| ---- | ----------------------------------------------------------------------------------------------- |
| 온도 | 25°C / 40°C / 60°C (히터 + PID 제어)                                                         |
| 클럭 | 5MHz ~ 스펙 한계 너머까지 스윕 (fR 초과 구간은 0x0B fast read — SPI 스펙에 더미 사이클 선반영) |
| 전압 | 3.3V → 저전압 방향 스윕 (레벨 시프터 경유)                                                     |
| 노화 | base: NOR 종단 마모 (0 → 인듀런스 한계 10만 회) / 확장: NAND 마모·반복 읽기                   |

각 조건에서: 쓰기 → 읽기 → 검증 → 윈도우 폭 측정 → (NAND) Fail Map + ECC → 기록(CSV) → 그래프.

**시료 운용 (W25Q64 ×10, base)**:

1. **전수 신품 측정** — 10개 모두. 기준 분포 + 개체 간 산포 확보 (비파괴이므로 비용 없음)
2. **종단 마모 3~4개** — 같은 칩을 체크포인트(예: 0 → 10k → 30k → 100k, 접미 확정 후 인듀런스 스펙 대비 재조정)마다 측정 → 칩 하나가 노화 곡선 하나를 생산
3. **블라인드 2개** — 마모 담당자가 횟수를 숨기고 마모, 측정자가 추정
4. **무마모 대조군 2개** — 실험 기간 내내 주기적 재측정으로 계측 장비 자체의 드리프트 검출
5. **예비 1~2개** — 소켓 손상·ESD·실수 대비

확장 시 동일한 배분 논리를 W25N 조달분에 적용한다.

**측정의 자기 검증 절차** (모든 본실험에 선행):

- **반복성 프로토콜** — 같은 칩을 재장착 포함 반복 측정하여 윈도우 폭의 재현 편차를 정량화한다.
  이 편차(측정 노이즈)가 예상 노화 이동량보다 충분히 작아야만 노화 실험이 성립한다.
  노이즈 플로어를 확립하기 전에는 마모를 시작하지 않는다.
- **블라인드 검증** — 마모 담당자가 마모 횟수를 숨긴 칩을 측정자가 추정하고,
  사전에 커밋된 정답과 대조해 오차 ±X%를 산출한다. 추정 성능을 사후 해석이 아니라
  사전 등록된 기준으로 검증한다.

### 기대 결과물

1. 온도·전압에 따른 **유효 윈도우 축소 곡선**
2. 전압-클럭 평면의 **슈무 플롯** (온도별로 여러 장)
3. 마모 횟수에 따른 **비트 에러 증가 곡선과 Fail Map**
4. 마모 횟수에 따른 **지우기 시간 증가 곡선** (노화 지표)
5. 같은 모델 칩 여러 개의 **개체 간 여유 분포** — 개체 산포(σ)와 노화 이동량(Δ)을 분리해야 교정 곡선을 선이 아닌 띠(band)로 정직하게 보고할 수 있다
6. 블라인드 수명 추정의 **오차 정량화 (±X%)**

## 9. 마일스톤

**욕조 곡선 마일스톤 (중간보고서, 게이트 순서 고정 — 상세는 `docs/milestone-bathtub.md`)**

1. **G0 루프백 욕조 곡선** — 칩·보드 없이 FPGA 단독으로 자기 배선의 곡선 실측. 최대 기술 리스크(MMCM 위상 제어)를 최우선 소진
2. **G1** cocotb + Winbond 모델 회귀 PASS — G0과 병렬 트랙(소유자·의존성 분리), 둘 다 PASS 후 G2 진입
3. **G2** 실칩 저클럭 브링업 (JEDEC ID)
4. **G3 실칩 욕조 곡선 1장** — 이 그래프가 프로젝트 전체 주장의 증명
5. **G4 반복성 정량화** — 재장착 포함 재현 편차. "측정이 노이즈가 아니라 신호"라는 증거

중간보고서 게재분 = 루프백 곡선 · 실칩 곡선 · 반복성 · 몬테카를로(위상당 읽기 횟수 N의 근거) · cocotb 검증 증거.
온도·전압 축은 의도적으로 이 마일스톤에서 제외 — 히터·레벨 시프터(전용 보드)가 전제라 일정 도박이 되기 때문.

**중간보고서 이후 (base — NOR로 완결)**: 온도 2점 윈도우 비교 → 전압 축·슈무 자동화 → **G5 마모 개시 게이트(아래)** → NOR 종단 마모(윈도우 폭 + 소거 시간 이중 지표) → 교정 곡선(band) → 블라인드 수명 추정, 오차 ±X% 산출

**G5 마모 개시 게이트** — 마모는 이 프로젝트에서 유일한 비가역 행위이므로 게이트를 둔다: ① 10개 전수 신품 측정 완료, ② 반복성 노이즈 플로어가 예상 노화 이동량 대비 충분히 작음을 수치로 확인, ③ 체크포인트 계획(마모 범위: 섹터/블록/칩 단위 포함)과 블라인드 정답 사전 커밋 완료. 셋 모두 충족 전에는 어떤 칩도 마모하지 않는다.

**확장 (NAND — 조달·일정 충족 시)**: W25N 실에러 측정 — Fail Map, ECC 정정 한계, 3종 지표 체계, on-die ECC 제어

**확장 (제조사별 비교 — 조달 시)**: 동일 스펙·핀 호환 SPI NOR를 제조사별(Winbond W25Q64 외 GigaDevice GD25Q64, Macronix 등 동급 칩)로 확보하여 동일 프로토콜로 측정한다. 신품 윈도우 분포·노화 곡선이 제조사 간에 어떻게 다른지 비교 — 측정 플랫폼이 특정 칩이 아니라 SPI NOR 일반에 성립함을 보이는 일반화 축이다.

> 원칙: 계획 10페이지보다 실측 그래프 1장이 강하다.

## 10. 핵심 철학 (요약)

1. **판정이 아니라 계측.** PASS/FAIL은 1비트짜리 답이다. 우리는 그 안에 숨어 있던 연속적인 "여유"를 꺼내 숫자로 만든다.
2. **에러를 기다리지 않고, 경계까지 다가간다.** 측정 장비 쪽의 타이밍을 움직여 여유를 직접 잰다. 그래서 정상 칩에서도 실험이 성립한다.
3. **인터페이스의 거동은 칩 내부 상태의 센서다.** 윈도우 폭, 동작 시간의 변화는 온도·노화가 남기는 지문이다.
4. **주장과 데이터의 일치가 최우선.** 만든 점수 대신 단위가 있는 측정값을, 과장 대신 명시된 범위를.
5. **자기 측정을 먼저 의심한다.** 반복성으로 측정 노이즈 플로어를 확립한 뒤에만 노화 신호를 주장하고, 수명 추정은 블라인드로 검증한다. 장비의 신뢰성(Fault Injection, oracle 3종)부터 증명한 뒤 칩을 논한다.
